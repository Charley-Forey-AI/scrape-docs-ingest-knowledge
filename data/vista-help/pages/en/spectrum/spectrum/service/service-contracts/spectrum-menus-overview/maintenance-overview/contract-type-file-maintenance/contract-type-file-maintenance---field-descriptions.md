# Contract Type File Maintenance - Field Descriptions

Use the table below for reference when completing the fields on this screen.
Fields/Buttons
Descriptions

Type code
Enter a unique code for the contract
 type. This code will be used when individual service contracts are set up to
 specify what type of contract the client has (for example, full-service, labor,
 and so forth). A lookup window is available for viewing existing types.
Description
Enter a full description of the
 contract type code entered above.
Billing source
Select whether the service contract is
 to be billed through Service
 Contract or Work
 Order.If Work Order is selected, the rest
 of the fields on this window will be display-only.

Default earned revenue basis

Default earned revenue basis
The selected earned revenue basis will
 serve as a default for new contracts, and will also be assigned when a proposed
 contract is activated. An additional earned revenue basis can optionally be
 assigned when editing individual contracts in Service Contracts to indicate
 that the deferred revenue schedule was manually adjusted.
Deferred revenue recognition

Allow (scheduled) deferred revenue processing for this contract type?
Select this checkbox to allow deferred
 revenue processing. When this checkbox is selected, the G/L account fields in
 this section will be enabled and must-enter. Once saved, you will not be
 allowed to clear this box in order to assure that revenue recognition for
 contracts will always have a valid G/L setup for the designated earned revenue
 basis in the contract.
Invoice distribution G/L account
This field defaults to the Service
 Contract Installation option for 'Default as billed invoice G/L account code.
 Direct cost G/L account codes are prohibited.
Deferred revenue G/L account
Direct cost G/L account codes are
 prohibited.
Revenue G/L account
Direct cost G/L account codes are
 prohibited.
'As billed' revenue recognition

Allow 'as billed' revenue recognition for this contract type
If the selected default earned revenue
 basis is 'As billed', this checkbox will be selected by default. If not,
 select this checkbox to recognize as billed revenue for this contract type.
 When this checkbox is selected, the Invoice distribution G/L account field
 will be enabled and must-enter. Once saved, you will not be allowed to clear
 this box in order to assure that revenue recognition for contracts will always
 have a valid G/L setup for the designated earned revenue basis in the contract.
Invoice distribution G/L account
This field defaults to the Service
 Contract Installation option for 'Default as billed' invoice G/L account code.
 You will be prevented from entering a 'direct cost' G/L account code (job,
 equipment or work order).
