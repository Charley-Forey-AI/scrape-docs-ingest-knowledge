# W/O Processing Date Change

This screen is used to view or change the processing date. The default is the current date, but the date can be changed here if desired. Note that the date displays in two formats: numerically on the left side of the screen, and on a calendar on the right side of the screen.
Field
Description

Current settings

Processing date
The current processing date displays.
 This is the date that will be used by the software during Work Order Entry. To
 make changes, type the date in the format MMDDYY; do not enter slashes as they
 will be automatically inserted by the software. This date will also be
 displayed on the calendar at the right. It is also possible to change the date
 by clicking the date on the calendar. If the new date is
 before the Minimum Processing Date or after the Maximum Processing Date
 established by the software administrator during Installation procedures, an
 error message will occur:
 DATE ENTERED IS NOT
 WITHIN CURRENT MINIMUM AND MAXIMUM DATE RANGE.
To
 leave the processing date as it is displayed, click the Reset button to restore
 present setting.

Fiscal year
The fiscal year in which the specified
 processing date is contained displays.
Period
The period number in which the
 specified processing date is contained (number 1-13) displays.
Period date
The starting and ending dates of the
 period in which the specified processing date is contained displays.
Allowable entry range

Minimum date
The first allowable processing date
 displays in this field. This date is set by the software
 administrator on the Installation Processing Date Change screen.


Maximum date
The last allowable processing date
 displays in this field. This date is set by the software
 administrator on the Installation Processing Date Change screen.
