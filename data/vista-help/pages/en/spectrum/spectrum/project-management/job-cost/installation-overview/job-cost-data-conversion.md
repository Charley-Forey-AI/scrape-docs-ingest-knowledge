# Job Cost Data Conversion

Learn how to setup Job Cost Data conversion in your Job
 Cost module.
It is important that these steps be performed in the order
 listed.

1. Complete all four tabs of the Job Cost Installation screen.

1. Set the processing date to the conversion date using
 Processing Date
 Change.

1. Enter all cost types (for example, L, M, S, O, and E) using Cost Type >  File Maintenance.

1. Print the cost types and check for accuracy.

1. Enter all standard phases using Standard Group / Phase
 Maintenance.

1. List all standard phases and check for accuracy.

1. Enter overhead allocation information, if applicable,
 using Overhead Allocation
 Maintenance.

1. Print the overhead allocations and check for
 accuracy.

1. Set up tax classification codes using Tax Classification File
 Maintenance.

1. If standard jobs are performed, enter file information
 for each standard job using Jobs. If no standard jobs are performed, proceed to step 13.

1. If standard jobs are performed, list the standard jobs
 and check for accuracy.

1. Enter all phases for the standard jobs using
 Job Phases.Tip: If Project Setup is present on the system,
 some users find it easier to set up phases and estimates by adding information in
 the Project Setup > Job Setup screen, then transferring to Job Cost.

1. List all phases for the standard jobs and check for
 accuracy.

1. Enter Quick Job Setup information, as needed, using Quick Job Setup.

1. Enter all jobs in progress as of the conversion date
 using Jobs.

1. Print all jobs in progress as of the conversion date
 and check for accuracy.

1. Enter the phases for all jobs in progress as of the
 conversion date using Job
 Phases.

1. List the phases for all jobs in progress as of the
 conversion date and check for accuracy.

1. Confirm that there are no job cost transaction entries
 in progress in the Cost Transaction
 Report or Job Cost G/L
 Detail Report.

1. Ensure that the Job Cost transactionscheckbox is
 not selected on the General Ledger
 Installationscreen.

1. Enter cost-to-date figures as of the end of the prior
 fiscal year for all jobs using Job Cost
 Transaction Entry.

1. Enter year-to-date cost figures for all jobs as of the
 conversion date using Job Cost
 Transaction Entry.

1. Print the Job Cost Transaction Report and
 check for accuracy. If correct, update. If not, make necessary changes before
 updating.

1. Print the Job Cost G/L Detail Report and
 check for accuracy. If correct, update. If not, make necessary changes before
 updating.

1. Clear week-to-date, month-to-date and year-to-date
 figures for job cost using Clear WTD,
 MTD, YTD Totals.

1. On the General Ledger Installation screen, select the Job Cost checkbox so that these
 figures will post to the General Ledger.

1. Set the processing dates for the new period using
 Processing Date
 Change.

1. If the Accounts Receivable module is installed, Job
 Cost conversion is complete; billing information should be entered during Accounts
 Receivable conversion. If the Accounts Receivable module is not installed, proceed to
 step 30.

1. If the Accounts Receivable module is not installed,
 enter the billings-to-date for all current jobs using Billing Transaction Entry.

1. Print the Billing Transaction Report and
 check for accuracy. If the information is correct, update.
