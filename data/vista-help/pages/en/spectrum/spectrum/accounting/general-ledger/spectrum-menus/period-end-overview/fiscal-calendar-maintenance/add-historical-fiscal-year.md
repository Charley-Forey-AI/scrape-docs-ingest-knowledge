# Add Historical Fiscal Year

Use this window to add previous years to the beginning of
 the fiscal calendar. If you already have historical data on your system, simply add those
 years in this screen to view the records and access the data for reports.
This window will default dates and period labels for the prior year,
 although you may edit the period labels.
Note: Users with historical data dating back prior to the
 release of v11 (2002) may not be able to access some data if it was previously purged from
 the software.
Fields
Descriptions

Fiscal periods

Fiscal year
The previous fiscal year defaults (based on the existing
 fiscal years). No changes are allowed.

# Periods
Enter the number of periods that you want to include in
 this fiscal year (the number defaults from then number of periods used in
 the existing fiscal years, but this number may be changed). Fiscal calendars
 can include anywhere from 1 to 13 periods; this allows companies to operate
 within a non-standard fiscal year.
Note: If fewer than 12 periods / 4 quarters comprise
 your fiscal calendar, then the end date of the current quarter is the last
 period that does exist in the calendar.

Start date
The starting date of the first period of this fiscal year
 defaults based on the pattern established for the earliest existing fiscal
 year. Press Enter
 to accept this date or use the Date Change window to select a different
 date.

Grid

#
The number of periods that comprise this fiscal year
 display. Fiscal calendars can include anywhere from 1 to 13 periods; this
 allows companies to operate within a non-standard fiscal year.

Period
The period labels display, and may be changed. Period
 labels are included on various reports and screens throughout Spectrum, and
 functions using these labels will be changed to reflect any changes made
 here.

Start date
The starting dates of the periods in this fiscal year
 default. The first period Start date is the same day as the Start date of the fiscal
 year, and each subsequent Start date is one day after the End date of the previous
 period. No changes are allowed.

End date
The end dates of the periods in this fiscal year default
 (based on the end dates used for established fiscal years). End dates may be
 changed, but they must be later than the previous period end date and two
 days before the next end date. Each time a new end date is entered, the
 software automatically adjusts the start date of the next period
 accordingly.
