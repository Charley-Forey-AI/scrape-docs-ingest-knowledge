# Detail Billing History Report - Field Descriptions

Use the table below for reference when completing the fields on this screen.
Fields/ButtonsDescriptions
Selections
Job numberPress Enter to print all jobs, or double-click in this field to select from a list of valid job numbers.Note: When printing an entire billing history for a selected job,
 use the Single Job-to-date
 Recap Crystal-Link option in the Report Setup window. This
 report format places the overhead, markup, and so forth for all bills at the
 end of the report rather than sequentially in the report, which enhances the
 overall readability of the detail billing history for a selected job.

Billing #Enter the number of the bill to print, or press Enter to print the largest posted bill number stored in the billing history file for each job. When a specific job is selected, enter ALL to print the history report with all posted billings for the job, or double-click to select from a list of posted bill numbers.
If the billing number exceeds 999, the billing number will automatically be converted to the alpha equivalent (Example: A00 = 1000).

Job notes?Select this checkbox to include remarks from the Job File Maintenance screen on this report.Note: Large sets of notes may not fully display on this
 report.

FormatSelect the report format you want to print:

- Detail with markup shown

- Detail with no markup display

- Single job: job-to-date recap

- Detail CSV - export as a CSV file for deeper analysis. Report headers appear only once, at the top of the CSV file.

Sort By
Phase / cost typeThe report will be sub-sorted by Job, Bill #, Phase, Cost type, Sequence.
Phase / cost type with group subtotalsThe report will be sub-sorted by Job, Bill #, Major/Minor group, Phase, Cost type, Sequence.
Cost type / phaseThe report will be sub-sorted by Job, Bill #, Cost type, Phase, Sequence.
Cost type / phase with group subtotalsThe report will be sub-sorted by Job, Bill #, Cost type, Major/Minor group, Phase, Sequence.
Cost type / dateThe report will be sub-sorted by Job, Bill #, Cost type, Transaction date, Work date, Reference 1, Sequence.
The Reference 1 sub-sort option will vary based on the main code for the Transaction type:

- AP transaction types sort by Vendor

- PR transaction types sort by Employee

- G/L transaction types sort by G/L description

- EC transaction types sort by Equipment code

Cost type / codeThe report will be sub-sorted by Job, Bill #, Cost type, Reference 1, Transaction date, Phase, Sequence.
The Reference 1 sub-sort option will vary based on the main code for the Transaction type:

- AP transaction types sort by Vendor

- PR transaction types sort by Employee

- G/L transaction types sort by G/L description

- EC transaction types sort by Equipment code

Phase digitsThe report will be sub-sorted by Job, Bill #, Phase digits, Phase, Cost type, Sequence
Phase digitsIf you chose Phase digits as the sort option:

- specify the starting and ending job phase

- choose whether to add page breaks between phases
