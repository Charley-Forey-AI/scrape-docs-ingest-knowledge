# Download and run the executable on the reverse proxy server

A step in the process of setting up Viewpoint Remote Link
 (VRL) for your organization.
This page is part of the process outlined in [Enable Vista Remote Link (VRL) Access](/en/vista/vista/on-premises-deployments/access-vista---vrl-on-premises/about-vista-remote-link-vrl-on-premises/enable-vista-remote-link-vrl-access). Before you proceed
 with the actions on this page, you need to have taken preparatory actions.
Run the VistaRemoteLinkServer.xx.x.x.xxx.exe file on your Vista Reverse Proxy server. For details, see [Upgrade the Reverse Proxy Server](/en/vista/vista/on-premises-deployments/installations/major-release-installation/vista-servers/upgrade-the-reverse-proxy-server).
For more information, see [Vista Servers](/en/vista/vista/on-premises-deployments/installations/service-pack-installation/vista-servers).
Important: Before installing the executable on the Reverse Proxy
 server, please note the following:

- This server must be in a workgroup, not attached to a
 domain.

- You must use a system account, not a domain account, to perform the install;
 ensure that account has administrator access in order to perform the
 install.
