# Field Definitions: PR ArrearsPayback History Form

The following is a list of field descriptions for the PR
 ArrearsPayback History form. Many of the descriptions include links to other topics that provide additional information about or related to the topic.

## Employee

This field displays the employee number for the arrears/payback transaction.
If you are entering a manual transaction,
 enter the employee number. Press F4 for a list of employees. For more
 information, see [Manually Entering Arrears/Payback Transactions](/en/vista/vista/hr-and-payroll/payroll/payroll-processing/payroll/employee-arrears-australia/manually-enter-arrears-and-payback-records).

## Ded Code

This field displays the deduction code for the arrears/payback transaction.
If you are entering a manual transaction,
 enter the deduction code for the employee. Press F4 for a list of deduction codes. You can
 enter only deduction codes that have been set as subject to arrears (the Subject to
 Arrears box is checked in PR Deductions/Liabilities, Arrears/Payback tab).
 For more information, see [Manually Entering Arrears/Payback Transactions](/en/vista/vista/hr-and-payroll/payroll/payroll-processing/payroll/employee-arrears-australia/manually-enter-arrears-and-payback-records).

## Seq

This field displays the sequence number for the arrears or payback transaction.
If you are entering a manual arrears or
 payback transaction, enter N, New, or + to generate the next available sequence
 number.

## Date

This field displays the date of the arrears/payback transaction.
If you are entering a manual transaction, enter the transaction date. For more information, see [Manually Entering Arrears/Payback Transactions](/en/vista/vista/hr-and-payroll/payroll/payroll-processing/payroll/employee-arrears-australia/manually-enter-arrears-and-payback-records).

## Arrears Amount

This field displays the arrears amount for the transaction.
If you are entering a manual arrears
 transaction, enter the amount. When you save the transaction record, the system will update
 the Arrears History
 Total and Life to Date Balance display fields in
 the header section of the form. For more information, see [Manually Entering Arrears/Payback Transactions](/en/vista/vista/hr-and-payroll/payroll/payroll-processing/payroll/employee-arrears-australia/manually-enter-arrears-and-payback-records).

## Payback Amount

This field displays the payback amount for the transaction.
If you are entering a manual payback
 transaction, enter the amount. When you save the transaction record, the system will update
 the Payback History
 Total and Life to Date Balance display fields in
 the header section of the form. For more information, see [Manually Entering Arrears/Payback Transactions](/en/vista/vista/hr-and-payroll/payroll/payroll-processing/payroll/employee-arrears-australia/manually-enter-arrears-and-payback-records).

## PR Group

PR Group field in the PR Arrears/Payback History form.
This field is read only and displays the payroll group specific to the pay period and employee. This column only contains values for records that were added during payroll processing and posting.
Not used if you are entering a manual arrears or payback transaction.

## PR End Date

PR End Date field in the PR Arrears/Payback History form.
This field is read only and displays the payroll end date related to the pay period for the employee. This column only contains values for records that were added during payroll processing and posting.
Not used if you are entering a manual arrears or payback transaction.

## Pay Seq

Pay Seq field in the PR Arrears/Payback History form.
This field is read only and displays the pay sequence number specific to the pay period and employee. This column only contains values for records that were added during payroll processing and posting.
Not used if you are entering a manual arrears or payback transaction.

## Memo

Enter any additional information you want to record for this manual arrears/payback transaction.
