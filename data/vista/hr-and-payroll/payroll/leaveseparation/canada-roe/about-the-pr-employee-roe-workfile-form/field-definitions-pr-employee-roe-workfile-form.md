# Field Definitions: PR Employee ROE Workfile Form

The following is a list of field descriptions for the PR
 Employee ROE Workfile form. Many of the descriptions include links to other topics that provide additional information about or related to the topic.

## Begin/End

Begin and End date fields on the PR Employee ROE Workfile form.
Enter the beginning and ending date in a range of separation dates for which to create ROE records. The system will bring in employees who have a separation date (in PR Employees, Add'l Info tab) that falls within this range.

## Default ROE Date

Default ROE Date on the PR Employee ROE Workfile form, Header section.
Entry in this field is required to enable the Fill Grid button.
 Enter the default ROE date. The date entered
 here will be use to populate the ROE Date field for all employees added to
 the grid below.

## Contact First Name / Last Name

Contact First Name / Last Name fields on the PR Employee ROE Workfile form, Header section.
Enter the first and last name of the person in your organization who is available to provide information and/or clarification about this employee/record of employment to Service Canada.
 The information entered here will default for all employees added to the grid below. If you leave these fields blank, you must enter the contact name individually for each employee in the grid before you initialize the ROE records.

## Comments

Comments field on the PR Employee ROE Workfile form, Header section.
Enter specific details about the ROE record (i.e. clarification of circumstances, etc.). The information entered here will default for all employees added to the grid below. If you prefer to enter comments for each employee individually, leave this field blank.
Note: If you select an ROE reason of K00-Other or K15-Other / Canadian Forces - Queen's Regulations / Orders, you will be required to enter comments for each employee in the grid below before you can initialize ROE records.

## Recall Code

Recall Code drop-down on the PR Employee ROE Workfile form, Header section.
From the drop-down list, select the default recall code to use for all employee records being added to the grid. The recall code you select here will default for all employees added to the grid; therefore, you should only select a code if that code applies to all or most employee records being added to the grid.

- Y-Date of Recall - Select if employee(s) will be returning to work and you know the expected return date.

- N-Not returning - Select if employee(s) will not be returning to work.

- U-Unknown - Select if employee(s) will be returning to work, but you do not know the expected return date.

If you selected the Y-Date of Recall recall code, enter the date you expect the employee(s) to return to work. Otherwise, you should leave this field blank. The date specified here will default for all employee records added to the grid.

## Employee

Employee field on the PR Employee ROE Workfile form, Grid section.
If you populated this grid using the Grid Fill button above, this field defaults employees with a separation date within the separation date range specified in the Required Inputs for Fill Grid section above. Cannot be overridden. If you do not want to generate an ROE record for an employee, you can either select the employee and select the Delete button in the toolbar or make sure the Process ROE checkbox is not selected for the employee.
If you are manually entering employees, enter the employee for which to generate an ROE record or press F4 to select from a list of valid employees.

## ROE Date

ROE Date on the PR Employee ROE Workfile form, Grid section.
Entry in this field is required before you can initialize ROE records.
This field defaults from the Default ROE
 Date field above (Required Inputs for Fill Grid section).
Accept the default or enter the ROE date for this employee.
Note: During ROE generation later, you will use the ROE Date for the purpose of selecting ROE records for inclusion in an XML file (for submission to Service Canada) or for printing ROE paper reports.

Select this checkbox if this employee record is ready for initialization. Do not select this checkbox if you are not ready to initialize this employee record.
Note: When you select the
 Process all ROE Records
 checkbox in the header section, the
 Process ROE
 checkbox becomes selected for all employee records in the grid. When you unselect the
 Process all ROE Records
 checkbox in the header section, the
 Process ROE checkbox
 becomes unselected for all employee records in the grid.

## Error Message

Error message field on the PR Employee ROE Workfile form, Grid section.
This field is only populated if you select the Initialize ROE button and the selected record could not be initialized due to missing or incorrect data, or if a duplicate record exists in PR Record of Employment. The error message will indicate the exact reason why the record could not be created. Once you correct the errors, you can reattempt to initialize the ROE record by selecting the Initialize ROE button again.
