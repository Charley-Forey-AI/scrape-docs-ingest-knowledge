# Create / Submit WA State Paid Family / Medical Leave Reports Using Aatrix

You can use PR Aatrix Report Selection to create and submit your WA Paid Family/Medical Leave reports.
 You must have Aatrix installed on your local workstation. For more information, see [Enable Aatrix on a Workstation](/en/vista/vista/hr-and-payroll/payroll/quarteryear-end-reporting/aatrix-reporting/enable-aatrix-on-a-workstation) and [About Regulatory Reporting Using Aatrix](/en/vista/vista/hr-and-payroll/payroll/quarteryear-end-reporting/aatrix-reporting/about-regulatory-reporting-using-aatrix).
The following details how to create and submit both the WA Paid Family Leave and WA Paid Family Leave - Employer reports using Aatrix:

1. From the main menu, select Payroll > Programs > PR Aatrix Report Selection.

1. In the Tax Year field, enter the reporting tax year.

1. From the Federal/State drop-down (to the right of the tax year), select WA. The Form Type field displays the regulatory forms relevant to the selected state

1. Select WA PFML FORM from the report list. The Date Filter section displays a Period drop-down.

1. From the Period drop-down, select the reporting quarter.

1. Select Continue to launch the Aatrix wizard.

1. Follow the prompts in the Aatrix wizard to complete the process.Note: Things to note as you proceed through the wizard:

- On the State & Local Tax Items page, you will need to add local tax items for both the employee and employer WA Paid Family Leave tax items (select the Add button to the right of the Local Tax Items section).For the employee tax item, set the State to WA and the Tax Name to Paid Family Leave.
For the employer tax item, set the State to WA and the Tax Name to Paid Family Leave - Employer.

- The PFML Employer Reference Number (first page of report) does not automatically populate; you will need to enter this number manually.

- You must resolve all areas highlighted in red before you can file the report, even if those areas are not related to the Family Leave report. If the red highlighted fields contain zero values and the zero values are correct, you must enter 0.00 in each field to validate the zero value. Otherwise, enter the correct value.
