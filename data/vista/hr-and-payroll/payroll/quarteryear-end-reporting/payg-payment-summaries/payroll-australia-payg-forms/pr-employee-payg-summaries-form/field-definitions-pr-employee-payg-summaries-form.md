# Field Definitions: PR Employee PAYG Summaries Form

The following is a list of field descriptions for the PR
 Employee PAYG Summaries form. Many of the descriptions include links to other topics that provide additional information about or related to the topic.

## Employee

Employee field on the PR Employee PAYG Summaries form.
Enter the employee to work with or press
 F4
 to select from a list of employees that have already been initialized for the selected tax
 year.
Note: If you need to enter a new employee (one that has not been initialized for the selected tax year), you must first add the employee to the Employees tab of [PR PAYG Summary Process](/en/vista/vista/hr-and-payroll/payroll/quarteryear-end-reporting/payg-payment-summaries/payroll-australia-payg-forms/pr-payg-summary-process-form).

## Summary #

Summary # field on the PR Employee PAYG Summaries form, PAYG INB Summaries tab.
Summary records are added to this grid automatically when you run the Generate Amounts process (by selecting Generate Amounts in PR PAYG Summary Process). The system creates a summary record for each reporting period found for the employee in the Period of Reporting table (PAYG Reporting Periods tab in PR Employees).
 You should never need to manually add summaries here and it is strongly recommended that you do not. If you manually enter Summaries using this form, care should be taken to avoid entering a Begin Date / End Date range that conflicts with other summaries in the same tax year or with existing Period of Employment records maintained in PR Employee setup. To do so could lead to ambiguous results reported to the ATO.
 For existing summary records, this is a system-generated number that is based on the PRCo, Tax Year, and Employee and cannot be changed.
 If you are adding a new summary record, enter N or + and the system will generate the summary number for you.
Note: This field is disabled once you close the tax year (i.e. set the PAYG Tax Year Open/Closed drop-down on the Info tab to PAYG Tax Year Closed).

## Period Begin Date

Period Begin Date field on the PR Employee PAYG Summaries form, PAYG INB Summaries tab.
Summary records are added to this grid automatically when you run the Generate Amounts process (by selecting Generate Amounts in PR PAYG Summary Process). The system creates a summary record for each reporting period found for the employee in the Period of Reporting table (PAYG Reporting Periods tab in PR Employees).
 You should never need to manually add summaries here and it is strongly recommended that you do not. If you manually enter Summaries using this form, care should be taken to avoid entering a Begin Date / End Date range that conflicts with other summaries in the same tax year or with existing Period of Employment records maintained in PR Employee setup. To do so could lead to ambiguous results reported to the ATO.
For existing summary records, this date is equal to the Period Start Date for the corresponding period of reporting record in PR Employees (PAYG Reporting Periods tab) and cannot be changed.
 If you are adding a new summary record, you must enter a period begin date that does not conflict with either the existing summary reporting period date ranges or the existing period of reporting date ranges ( in PR Employees, PAYG Reporting Periods tab).
Note: This field is disabled once you close the tax year (i.e. set the PAYG Tax Year Open/Closed drop-down on the Info tab to PAYG Tax Year Closed).

## Income Type

Income Type drop-down on the PR Employee PAYG Summaries form, PAYG INB Summaries tab.
For summaries generated via PR PAYG Employee
 Generate Amts, this field defaults the Income Type specified for the
 corresponding reporting period in PR Employees (PAYG Reporting Periods tab). You should not
 need to change the default.
 If you are manually adding a new summary record, select the income type or accept the defaulted value. Income types are as follows:

- S - Salary or wages (default) - Employee is a full or part-time employee receiving a regular salary or hourly wages.

- H - Working holiday maker - Employee is a working holiday maker.

Note: This field is disabled once you close the tax year
 (that is, set the PAYG Tax Year Open/Close drop-down in PR PAYG Summary Process to PAYG Tax Year Closed).

## Amended Summary

Amended Summary checkbox on the PR Employee PAYG Summaries form, PAYG INB Summaries tab.
The checkbox only applies when printing the PR PAYG Payment Summary - Individual Non-Business report.
Select this checkbox if you are generating an amended PAYG individual non-business summary for this employee after previously submitting a summary. The system indicates the amended status clearly on the PR PAYG Payment Summary - Individual Non-Business report, per ATO requirements.
Do not select this checkbox if you are generating an original PAYG individual non-business summary for this employee.
Note: This field is disabled once you close the tax year
 (i.e. set the PAYG
 Tax Year Open/Closed drop-down on the Info tab to PAYG Tax Year
 Closed).

## Include in E-File

Include in E-File checkbox on the PR Employee PAYG Summaries form, PAYG INB Summaries tab.
Use of this checkbox only applies if you will be generating an e-file for submission of PAYG summaries to the Australian Taxation Office (ATO).
This checkbox initially defaults as selected. Accept the default setting if this summary record should be included in the ATO e-file.
Clear this checkbox if this payment summary record should not be included in the ATO e-file.
Note: The system will also clear or select this checkbox
 for this employee when you edit the settings in [PR ETP/PAYG E-File Set/Unset](/en/vista/vista/hr-and-payroll/payroll/quarteryear-end-reporting/payg-payment-summaries/payroll-australia-payg-forms/pr-etppayg-e-file-unsetset-form) (accessed via Tasks >  Unset/Set
 'Include in E-File' checkboxes from the [PR ETP Summary Process](/en/vista/vista/hr-and-payroll/payroll/quarteryear-end-reporting/payg-payment-summaries/payroll-australia-payg-forms/pr-etp-summary-process-form) toolbar and the [PR PAYG Summary Process](/en/vista/vista/hr-and-payroll/payroll/quarteryear-end-reporting/payg-payment-summaries/payroll-australia-payg-forms/pr-payg-summary-process-form) toolbar).
Note: This field is disabled once you close the tax year
 (i.e. set the PAYG
 Tax Year Open/Closed drop-down on the Info tab to PAYG Tax Year
 Closed).

## E-File Amendment Indicator

E-File Amendment Indicator drop-down on the PR Employee PAYG Summaries form, PAYG INB Summaries tab.
The option selected here will only be used if
 you also selected the Include in E-File checkbox.
From the drop-down menu, select one of the following:

- O-Original (Default) - Select this
 option if this is the first (original) submission of this payment summary to the ATO.
 The system will flag the summary record in the XML e-file as Original.

- A-Amended - Select this option if
 this payment summary record has been previously submitted to the ATO, and is now
 being resubmitted with amended values. The system will flag the summary record in the
 XML e-file as Amended.
Note: This field is disabled once you close the tax year (i.e. set
 the PAYG Tax
 Year Open/Closed drop-down on the Info tab to PAYG Tax Year
 Closed).

## ATO Item

ATO Item field on the PR Employee PAYG Summaries form, ATO / Super Amounts tab.
ATO items applicable to this employee/summary record are added to this grid automatically when you run the Generate Amounts process (by selecting Generate Amounts in PR PAYG Summary Process).
 You will typically not need to add new items here, since the system will pull in all items applicable to the employee based on the items defined for the tax year in PR PAYG Summaries (ATO / Super Items tab) and the amounts found for each summary reporting period in PR Employee Accumulations. However, if you choose to do so, items must be valid for the tax year and not already exist for the selected summary.
 If you are adding a new item, enter the type of item for this record or press F4 to select from a list of valid item types.
Note: This field is disabled once you close the tax year (i.e. set the PAYG Tax Year Open/Closed drop-down on the Info tab to PAYG Tax Year Closed).

## Amount

Amount field on the PR Employee PAYG Summaries form, ATO / Super Amounts tab.
ATO items are added to this grid automatically when you run the Generate Amounts process (by selecting Generate Amounts in PR PAYG Summary Process). The amount shown here is the summed amount for all earnings, deduction, and liability codes associated with the item for the selected summary's reporting period.
Note: You can edit the item amount if necessary, as long as the Tax Year is not closed (i.e., the PAYG Tax Year Open/Close drop-down in PR PAYG Summary Process is not set to PAYG Tax Year Closed); however, it will not change the amounts elsewhere in the system. Additionally, if you regenerate employee amounts, the system will overwrite any changes you make here.
If you are adding a new ATO item, enter the total amount for this item.

## LSA Type

LSA Type field on the PR Employee PAYG Summaries form, ATO / Super Amounts tab.
This field is only enabled when entering new items with an ATO Item code of LSA.
ATO items are added to this grid automatically when you run the Generate Amounts process (by selecting Generate Amounts in PR PAYG Summary Process). For items with an LSA item code, this field defaults based on the ATO Category assigned to the earnings code recorded for this employee in PR Employee Accumulations and cannot be changed. For example, if the employee has accumulated earnings (in PR Employee Accumulations) for an earnings code with an ATO Category of LSAR (Leave, Lump Sum A Type R), this field will default as R-Type R.
If you are adding a new item, from the drop-down, select the LSA type:

- T-Type T (Default) - Leave Payment when termination is not for reason of redundancy

- R-Type R - Leave Payment when termination is for reason of redundancy
Note: This field is disabled once you close the tax year (i.e. set the PAYG Tax Year Open/Closed drop-down on the Info tab to PAYG Tax Year Closed).

## Name of Organisation

Name of Organisation field on the PR Employee PAYG Summaries form, Misc Amounts tab.
This field is only enabled for items with an ATO Item code of F (fees) or G (giving amounts).
Misc items are added to this grid automatically when you run the Generate Amounts process (by selecting Generate Amounts in PR PAYG Summary Process).
Enter the name of the organisation to which the fees or workplace giving amounts were paid.
Note: This field is disabled once you close the tax year (i.e. set the PAYG Tax Year Open/Closed drop-down on the Info tab to PAYG Tax Year Closed).
