# Set a Labor Rate Override

You can set a labor rate override.

1. Restrict the labor rate by employee, craft, class, earn type, factor, and/or shift by selecting the Restrict by Employee, Restrict by Craft, Restrict by Class, Restrict by Earn Type, Restrict by Factor, and/or Restrict by Shift checkboxes. Leave the checkbox blank to indicate “all.”  Depending on the selected checkboxes, the PRCo, Employee, Craft, Class, Earn Type, Factor and Shift fields are enabled.

1. Indicate the PR company, employee, craft, class, earn type, factor, or shift in the PRCo, Employee, Craft, Class, Earn Type, Factor, and/or Shift fields, as necessary. Note: For example, a labor category can be restricted by a specific earnings type, but can include all earning factors and shifts. Any combination of the three restricting options can be used.

1. Select a rate option from the Rate Option drop-down: C-Actual Costs, F-Earnings Factor, or R-Labor Rate. Note: If you select C-Actual Costs, labor is billed using the actual costs posted to JC. If you select R-Labor Rate, labor is billed using the rate specified for the labor category. If you select F-Earning Factor, labor is billed using the earnings factor posted to JC multiplied by the labor category’s rate.

1. Enter the new rate in the New Rate field. Note: Update old rates for all entries in the grid by selecting File > Move New Rates to Old. When this option is selected, the Old Rate field is
 automatically updated with the values in the New Rate fields. The New Rate
 fields do not change and must be manually updated.
