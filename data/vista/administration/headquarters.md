# Headquarters

The Headquarters module is the foundation for all modules on the system.
Each company that you set up in other modules must be set up in HQ first. In addition, HQ is used to maintain system-wide information that affects how information is processed in all of the Accounting modules, such as the various codes (states, units of measure, earn types, frequency codes, etc.) used by the accounting modules.
Headquarters contains programs that access highly sensitive information to which most users should not have access. It is for this reason that you should carefully consider who should be allowed to set up this information, and who should be allowed to change or delete that information once it is set up.
For more information, select any of the following topics:

- [Setting Up Your First HQ Company ](/en/vista/vista/administration/headquarters/company-setup/forms-to-use-in-setting-up-your-first-hq-company)

- [Setting Up Additional HQ Companies](/en/vista/vista/administration/headquarters/company-setup/about-setting-up-additional-hq-companies)

- [Changing a company logo](/en/vista/vista/administration/headquarters/company-setup/changing-a-company-logo)

If you have the Workflow module, you can use the Implementation Checklists and Processing Logs for the HQ module. which are available in the Workflow module. To view the checklists and logs, use the WF Checklist Template form. For information on processing the checklists and logs, see the [Work Flow Overview](/en/vista/vista/system-tools/work-flow).
If you do not have the Workflow module, use the WF Templates report (VA Reports menu) to view or print the checklists and logs.
