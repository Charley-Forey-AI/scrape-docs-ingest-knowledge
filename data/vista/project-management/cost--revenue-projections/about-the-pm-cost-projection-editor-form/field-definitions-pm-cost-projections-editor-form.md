# Field Definitions: PM Cost Projections Editor Form

The following is a list of field descriptions for the PM Cost
 Projections Editor form. Many of the descriptions include links to other topics that provide additional information about or related to the topic.

## Buyout

The Buyout checkbox on the PM Cost Projection Editor form, Info tab.
Select this checkbox if the buy out is complete on a projection code or phase/cost type. This means the total commitments have been made through subcontracts, purchase orders, and/or material orders. The system automatically calculates the final values based on your projection method (> Actual Cost or Actual + Committed Cost fields).
 If you select this checkbox on a projection code, the system automatically selects the Buyout checkbox on all of the phases/cost types associated with the projection code.
 The selection in this field also applies to any linked cost types. Click here for information about linked cost types.
Note: If a row is highlighted in blue, this means a plugged value on a projection code or phase/cost type does not equal the calculated buyout value.

## % Complete - Units

The % Complete - Units field on the PM Cost Projection Editor
 form, Info tab.
This field is disabled if the Percent
 Complete checkbox is not selected on the PM Company Parameters (Projections
 tab, Columns For Entry section).
Use this field to override (plug) the percent complete for units. This is the
 actual units incurred divided by the projected final units.
When this field displays on the Grid tab of the PM Cost Projections form, the calculation
 includes all of the phase/cost types associated with the projection code. You can see
 these phases and cost types using the Phase/Cost Types tab.
Note: You can only make a change to this field if there are actuals posted to the
 selected phase/cost type. Actuals display in the Actual and
 Actual + Committed Cost columns.

For information about how plugged values are distributed, see the [ Spread Option](/en/vista/vista/project-management/setupmaintenance/pm-company-parameters-form/field-definitions-pm-company-parameters-form#ID-00023f71--en) F1 help.

## Remaining - Unit Cost

The Remaining - Unit Cost field on the PM Cost Projections Editor form, Info tab.
This field is disabled if the Remaining checkbox is not selected on the PM Company Parameters (Projections tab, Columns For Entry section).
Use this field to override (plug) projected remaining amounts.


 If using the ManDays option (last line), the value entered here will be
 multiplied by the Hrs/ManDay (shown above the entry grid) to determine
 the remaining hours. For example, if Hrs/ManDay is set to ‘8’, and you
 change the remaining man-days from ‘10’ to ‘15’, the remaining hours will
 be increased to 120 (15 x 8). All other man-day and hours values (i.e.
 over/under and final) are recalculated accordingly.
Plugged values are distributed based on the Spread Option defined in PM Company Parameters. For more information, see the [ Spread Option](/en/vista/vista/project-management/setupmaintenance/pm-company-parameters-form/field-definitions-pm-company-parameters-form#ID-00023f71--en) F1 help.

## Notes

Notes tab on the PM Cost Projection Editor form.
Use this field to enter notes on the projection. You can also enter notes at the projection code level using the Notes tab on the PM Cost Projections form.

### Spelling Check / Standard Notes

To spell check the text in this field, click the Spelling icon on the toolbar or select Tools > Spelling.
To add a Standard Note, right click the mouse while focus is in the field and select Standard Notes from the shortcut menu, which opens the Standard Note Copy window. Then enter the standard note to copy (or select from F4 lookup) and click OK. The system inserts the selected note into the field.

## Action

The Action drop-down on the PM Cost Projection Editor form, Projection Detail tab.
 Use this field to select the action that you would like to perform.

- Add - This option is automatically selected when creating new items and it cannot be changed.

-  Change - Select this option to make a change to detail lines that have already been processed.

-  Delete - Use this action to delete the detail item. This will eliminate the record from all future projection batches. Using the Delete () icon in the toolbar and Records menu will only delete the entry from the batch.

Note: The Projection Detail tab is only enabled when the Activate projection detail feature checkbox is selected in the PM Company Parameters form (Projections tab). The fields on the Projection Detail tab are enabled based on the JB cost type category associated with the selected cost type in (JC Cost Types, JC Cost Type Category field).

## UM

The UM field on the PM Cost Projection Editor form, Projection Detail tab.
Enter the unit of measure for this projection detail line. Initially defaults the unit of measure as follows:

- If you specified a budget code with a Costing Level of ‘Detail’ and a basis of ‘Hour’, defaults the Time UM specified for the budget code in JC Budget Codes.

-  If you specified a budget code with a Costing Level of ‘Detail’ and a basis of ‘Unit’, defaults cost type’s assigned unit of measure.

-  If you specified a budget code with a Costing Level of ‘Subtotal’ or ‘Total’, defaults as null. If you did not specify a budget code, defaults the cost type’s assigned unit of measure

Note: The Projection Detail tab is only enabled when the Activate projection detail feature checkbox is selected in the PM Company Parameters form (Projections tab). ## Unit Cost

The Unit Cost field on the PM Cost Projection Editor form, Projected Detail tab.
Enter the unit cost for this projection detail line.
If you selected an hour-based budget code in the Budget Code field, this field defaults a value as follows:

- Labor or Equipment cost type - Defaults based on cost/hour x Number Of field

-
Burden cost type - Defaults from the cost/hour

- Material, Subcontract, or Other cost type - Defaults as null.

The following applies if you selected a unit-based budget code in the Budget Code field:

-
If you selected a labor or equipment cost type (Cost Type field at the top of the form) and a Cost/Hour, this field defaults the
 Cost/Hour multiplied by the Number Of field.

-
If you selected a burden cost type, this field defaults the value entered in the Cost/Hour field.

-
If you selected a material, subcontract, or other cost type, and the Budget UM equals the cost type
 UM, this field defaults the unit cost set up on the budget code.

Note: The Projection Detail tab is only enabled when the Activate projection detail feature checkbox is selected in the PM Company Parameters form (Projections tab). The fields on the Projection Detail tab are enabled based on the JB cost type category associated with the selected cost type in (JC Cost Types, JC Cost Type Category field).

## Costs

The Cost field on the PM Cost Projection Editor form, Costs tab.
Use this field to filter the type of costs that display in the lower portion of the form. This will also affect the columns that display.

- Actual – Show all actual costs for the selected job/phase/cost type.

-  Committed – Show all committed costs for the selected job/phase/cost type.

-  Estimate – Show all estimated costs for the selected job/phase/cost type.

-  Projected – Show all projected costs for the selected job/phase/cost type.
