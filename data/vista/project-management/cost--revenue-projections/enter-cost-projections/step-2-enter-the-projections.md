# Step 2: Enter the projections

1. Group, filter, and sort the
 columns to quickly locate important projections, for example the projections where you
 have the most exposure (largest estimates or largest change orders).

1. You can enter projection
 information using the Grid tab on the PM Cost Projections form. The plugged values are
 distributed to the phases/cost types associated with the projection code based on how the
 PM module is set up (PM Company Parameters > Projections tab> Spread Option
 field). For more information, see [About Using the Spread Option for Cost Projections](/en/vista/vista/project-management/cost--revenue-projections/about-using-the-spread-option-for-cost-projections).

1. You can enter a projection using the detail form (PM Cost Projections Editor) or the Phase/Cost Type tab (PM Cost Projections). When you make a change to one, the other is immediately updated.If you enter a plugged value on a primary cost type, the system will automatically update the linked cost types.
For example assume that labor is a primary cost type and burden is a linked cost type. If you increase the
 Units Remaining
 on labor by 10%, the system will automatically increase the
 Units Remaining
 on burden by 10%.
 For information about linked cost types, see [About Using Linked Cost Types](/en/vista/vista/project-management/cost--revenue-projections/about-using-linked-cost-types) for information about linked cost types.
If the buyout is complete for a projection code or phase/cost type, select the Buyout checkbox. This means the total commitments have been made
 through subcontracts, purchase orders, and/or material orders. The system automatically
 calculates the final values based on your projection method.
If you check this box on a projection code, the
 system will also automatically check Buyout box on all of the phases/cost
 types associated with the projection code.
The selection in this field also applies to any linked cost types.Note: If you select the Buyout checkbox and
 a plugged value on a projection code or phase/cost type does not equal the calculated
 buyout value, the row will be highlighted in blue.

1. Optional: Enter
 the projection detail records using the Projection Detail tab on the PM Cost Projection
 Editor form. The Projection Detail tab on the PM Cost
 Projection Editor form will only display if the Activate projection detail feature box
 is checked (PM Company Parameters>Projections tab).
Press F1 in any field on the Projection
 Detail tab for detailed information about that field.

1. Optional: Use
 the Future Change Orders tab on the [PM Cost Projection Editor](/en/vista/vista/project-management/cost--revenue-projections/about-the-pm-cost-projection-editor-form) form to view the pending
 change order (PCO) items and approved change order (ACO) items that are associated with
 the selected project/phase/cost type, but have not been interfaced with the Job
 Cost module.
