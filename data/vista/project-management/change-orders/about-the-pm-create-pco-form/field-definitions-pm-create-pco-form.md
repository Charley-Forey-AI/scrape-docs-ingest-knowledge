# Field Definitions: PM Create PCO Form

The following is a list of field descriptions for the PM
 Create PCO form. Many of the descriptions include links to other topics that provide additional information about or related to the topic.

## PCO Type

Enter the PCO type. Must be a valid document type (set up in PM Document Types) with a category of Pending Change Order.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## PCO

Enter a PCO, up to 10 characters, or enter
 + to auto-generate the next
 sequential number available. Numbers will be generated based on the Auto-Generate Pending
 Change Orders using option designated in PM Projects (Project or
 Project/Type).
If you enter an existing PCO number, the PCO header fields will be disabled and cannot be changed.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Description

Enter a description of the pending change order. The value in this field can be up to 60 characters long.
By default this field will populate with the description of the RFI or project issue used to create the PCO.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Priority

Use this drop down to select the priority of the pending change order. You can use this field to sort and filter pending change orders in the PM module [Work Center](/en/vista/vista/system-tools/work-centers/about-the-project-management-work-center) and other places in the application.
By default the priority will be set to medium.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Estimate

Check this box if the pending change order impacts the budget of the project.
This field will populate with the default
 value set up on the document type selected in the PCO Type field, but it can be changed if
 necessary. Use the Pending Change Order section on the Info tab of the [PM Document Type](/en/vista/vista/project-management/setupmaintenance/pm-document-types-form) form to define this default value.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Subcontract

Check this box if the pending change order impacts a subcontract on the project. This will enable the subcontract fields on the Estimate/Purchase Details tab of [PM Pending Change Orders ](/en/vista/vista/project-management/change-orders/pm-pending-change-orders-form) once the PCO is created.
This field will populate with the default
 value set up on the document type selected in the PCO Type field, but it can be changed if
 necessary. Use the Pending Change Order section on the Info tab of the [PM Document Type](/en/vista/vista/project-management/setupmaintenance/pm-document-types-form) form to define this default value.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Purchase

Check this box if the pending change order impacts a purchase order on the project. This will enable the PO fields on the Estimate/Purchase Details tab of [PM Pending Change Orders ](/en/vista/vista/project-management/change-orders/pm-pending-change-orders-form) once the PCO is created.
This field will populate with the default
 value set up on the document type selected in the PCO Type field, but it can be changed if
 necessary. Use the Pending Change Order section on the Info tab of the [PM Document Type](/en/vista/vista/project-management/setupmaintenance/pm-document-types-form) form to define this default value.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Contract

Check this box if the pending change order impacts a contract item on the project. This box should be checked when you use the lower portion of this form to enter contract change amounts.
This field will populate with the default
 value set up on the document type selected in the PCO Type field, but it can be changed if
 necessary. Use the Pending Change Order section on the Info tab of the [PM Document Type](/en/vista/vista/project-management/setupmaintenance/pm-document-types-form) form to define this default value.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Notes

Check this box to copy notes entered on the Notes tab for the RFI to the pending change order.

- If checked for the PCO header, the RFI notes will be copied to the Notes tab in the pending change order header.

- If checked for the PCO Item, the RFI notes will be copied to the Notes tab for the pending change order item.

Leave this box unchecked to exclude RFI notes when creating the pending change order. Notes will not be copied to the pending change order header (if unchecked at the PCO header level) or the pending change order item (if unchecked at the PCO Item level).

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Response

Check this box to copy notes entered on the Response tab in the RFI header to the pending change order.

- If checked for the PCO header, the RFI response will be copied to the Notes tab in the pending change order header.

- If checked for the PCO Item, the RFI response will be copied to the Notes tab for the pending change order item.

Leave this box unchecked to exclude the RFI response (header level) when creating the pending change order. Response will not be copied to the pending change order header (if unchecked at the PCO header level) or the pending change order item (if unchecked at the PCO Item level).
Note:This field only applies when you are creating a PCO from an RFI, so it is disabled when this form is launched from [PM Issues ](/en/vista/vista/project-management/issuessubmittalsother-documents/project-issues/pm-issues-form).

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Information Requested

This field only applies to pending change orders created using an RFI.
Check this box to copy the notes entered on the Information Requested tab in the RFI header to the pending change order.

- If checked for the PCO header, the RFI ‘information requested’ notes will be copied to the Notes tab in the pending change order header.

- If checked for the PCO Item, the RFI ‘information requested’ notes will be copied to the Notes tab for the pending change order item.

Leave this box unchecked to exclude the RFI ‘information requested’ notes when creating the pending change order. Notes will not be copied to the pending change order header (if unchecked at the PCO header level) or the pending change order item (if unchecked at the PCO Item level).
Note:This field only applies when you are creating a PCO from an RFI, so it is disabled when this form is launched from [PM Issues ](/en/vista/vista/project-management/issuessubmittalsother-documents/project-issues/pm-issues-form).

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Attachments

Check this box to copy attachments for the RFI to the pending change order.
Leave this box unchecked to exclude RFI attachments when creating the pending change order.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## PCO Item

Enter the pending change order item, up to 10 characters, or enter ‘+’ to assign the next sequential item number.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Description

Enter the pending change order item description. The value in this field can be up to 60 characters long.
Initially this field's default value is the
 PCO description, but you can change it if needed.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Status

Enter the status for this change order item. Must be a valid status set up in PM Status IDs. Initially defaults the status assigned to the RFI.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Contract Item

Enter the contract item for this change order item. If you enter a contract item that does not already exist for the contract, the PM Contract Items form displays, allowing you to set up the new contract item. When finished, you are then returned to this screen.
If the change order item will not be using the same unit of measure and unit price as the contract item to which it is associated, it is suggested that you add a new contract item to ensure proper tracking of units and costs.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Fixed Amount

Check this box to enter a fixed contract
 amount for this PCO item. This will enable the Fixed Amount field, which is used to
 enter the fixed amount contract item.
If you check this field and enter a unit of
 measure that is not LS in the UM field, the Unitsand
 Unit
 Price fields are enabled.
Leave this box unchecked if you want to use
 calculated amounts.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Units

Enabled only if Fixed Amount Flag is checked and the UM is not LS.
Enter the number of units for this pending change order item.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## UM

Enter the UM for this pending change order item. Defaults the UM from the contract item assigned to this pending change order item. If you change the UM, a warning will display indicating that the PCO item unit of measure does not match the contract item’s UM. However, entry is allowed.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Unit Price

Enabled only if Fixed Amount Flag is checked and UM is not LS.
Enter the unit price for this pending change order item. Defaults the unit price specified for the contract item assigned to this pending change order item. If no unit price assigned to the contract item, defaults as 0.00.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Force Phase

Check this box to force all phases set up for this pending change order item (in PM Pending Change Orders) to use the contract item assigned to this pending change order item.
Leave this box unchecked if phases set up for this pending change order item should use their assigned contract items, regardless of whether they match the contract items assigned to the change order item.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Change in Days

Enter the number of days that this pending change order item changes the original contract completion date.
By default, this field populates with the value in the Days field in the Schedule/Cost Impact section on [PM Request For Information](/en/vista/vista/project-management/issuessubmittalsother-documents/rfis/about-the-pm-request-for-information-form) or [PM Issues ](/en/vista/vista/project-management/issuessubmittalsother-documents/project-issues/pm-issues-form).

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO

## Fixed Amount

Enter the contract change amount for this
 change order item - the value in this field will populate in the Fixed Amount
 field on the Info tab in the lower portion of [PM Pending Change Orders ](/en/vista/vista/project-management/change-orders/pm-pending-change-orders-form). Make sure that the Contract box at
 the top of the form is checked if you enter an amount greater than 0.00 - a message box
 will appear when you click OK if you do not.
Note: This field is only enabled when the Fixed Amount
 box is checked.
By default, this field will populate with the ROM (rough order of magnitude) price entered on the issue or RFI used to create the PCO.

- [PM Issues ](/en/vista/vista/project-management/issuessubmittalsother-documents/project-issues/pm-issues-form) -
 The ROM price is entered in the ROM Price field in the
 Schedule/Cost Impact section on the Info tab.

- [PM Request For Information](/en/vista/vista/project-management/issuessubmittalsother-documents/rfis/about-the-pm-request-for-information-form) - The ROM price is entered
 in the ROM
 Price field in the Schedule/Cost Impact section on the Info tab.

[](/en/vista/vista/project-management/change-orders/about-the-pm-create-pco-form)PM Create PCO
