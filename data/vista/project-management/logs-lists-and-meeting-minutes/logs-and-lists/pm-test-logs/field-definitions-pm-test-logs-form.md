# Field Definitions: PM Test Logs Form

The following is a list of field descriptions for the PM Test
 Logs form. Many of the descriptions include links to other topics that provide additional information about or related to the topic.

## Project

Enter a project or press F4 to select
 one from a list. This is the project that applies to the test log.

[](/en/vista/vista/project-management/logs-lists-and-meeting-minutes/logs-and-lists/pm-test-logs)PM Test Logs

## Test Type

Enter the type of test or press F4 to select one from a list.
Only active document types with a category of
 'Test Logs' can be selected. Document types are created and maintained using the [PM Document Types ](/en/vista/vista/project-management/setupmaintenance/pm-document-types-form) form, and they are set up as test types
 using the Document
 Category drop down.

[](/en/vista/vista/project-management/logs-lists-and-meeting-minutes/logs-and-lists/pm-test-logs)PM Test Logs

## Test Log

Create a new test log
When creating a new test log, you are creating
 a test log of the type selected in the Test Type field.
There are several ways to create a new test log using this field:

- Enter a "+" and then press TAB. The system will automatically
 create a new test log and assign it the next available number.

- Click the New Record icon () and then
 select the type of test log that you would like to create in the Test Type
 field. The system will populate the Test Log field with the next available
 number. Press TAB to accept the default and create the new test log.

- Enter a test log number that has not already been created for the
 type selected in the Test Type field. The test log number
 can be up to 10 characters long.

Note: The system will ignore alphanumeric numbers when
 determining the next number to assign.Select an existing test log

Enter an existing test log number or press F4 to select one from a list.

[](/en/vista/vista/project-management/logs-lists-and-meeting-minutes/logs-and-lists/pm-test-logs)PM Test Logs

## Description

Enter a description of the test log. The description can be up to 30 characters long.

[](/en/vista/vista/project-management/logs-lists-and-meeting-minutes/logs-and-lists/pm-test-logs)PM Test Logs

## Location

Enter the location of the test or press F4 to select one from a list.
Project locations are created and maintained using the [PM Project Locations ](/en/vista/vista/project-management/projects-and-contracts/projects/pm-project-locations-form) form.

[](/en/vista/vista/project-management/logs-lists-and-meeting-minutes/logs-and-lists/pm-test-logs)PM Test Logs

## Test Firm

Enter the firm performing the test or press
 F4
 to select it from a list. Firms are created and maintained using the [PM
 Firms ](/en/vista/vista/project-management/setupmaintenance/about-the-pm-firms-form)form.
Using the sort name
Tip: The sort name can be used instead of the firm number
 when selecting firms in PM module forms. Generally the sort name is the first several
 letters of the firm name.  For example when creating a subcontract in the [PM Subcontracts](/en/vista/vista/project-management/subcontracts/pm-subcontracts-form) form for 'Bryan Electrical', you can enter the sort name 'bryan'
 in the Vendor field instead of the firm number '10042'. The sort name of a firm is
 set up using the Sort Name field on the [PM Firms](/en/vista/vista/project-management/setupmaintenance/about-the-pm-firms-form) form.

[](/en/vista/vista/project-management/logs-lists-and-meeting-minutes/logs-and-lists/pm-test-logs)PM Test Logs

## Test Contact

Enter the test log contact - this is either
 the person performing the test or the person to contact regarding the test. This contact
 must be associated with the firm selected in the Test Firm field.
Contacts are created and maintained using the [PM Firm Contacts](/en/vista/vista/project-management/setupmaintenance/about-the-pm-firm-contacts-form) form.
Note: Once the contact is selected, their phone, fax, and
 email address information will display on the form.

[](/en/vista/vista/project-management/logs-lists-and-meeting-minutes/logs-and-lists/pm-test-logs)PM Test Logs

## Tester Name

Enter the name or initials of the person who performed the test. The tester name can be up to 30 characters long.

[](/en/vista/vista/project-management/logs-lists-and-meeting-minutes/logs-and-lists/pm-test-logs)PM Test Logs

## Status

Enter the status of the test or press
 F4
 to select one from a list.
Status codes are created and maintained using the [PM Status IDs](/en/vista/vista/project-management/setupmaintenance/pm-status-ids-form) form.

[](/en/vista/vista/project-management/logs-lists-and-meeting-minutes/logs-and-lists/pm-test-logs)PM Test Logs

## Notes

Enter any remarks or notes regarding this test log document.
Add a Standard Note
Standard notes allow you to insert frequently used text into some fields in the application. This text is created and maintained using the [HQ Standard Note](/en/vista/vista/administration/headquarters/standard-notes/about-the-hq-standard-note-form) form.
To insert a standard note into the field,
 right click the mouse while focus is in the field and select Standard Notes
 from the shortcut menu, which opens the Standard Note Copy window. Then enter the standard
 note to copy (or select from F4 lookup) and click OK. The system inserts the selected note
 into the field.
Spelling Check
 Click the Spelling icon on the toolbar or select Tools >  Spelling  to spell check the text in this field.
