# Create a New Issue

You can create project issues to track possible problems and issues that occur during the lifetime of a project.
 Once you create a project issue, you can use the Create and Send feature to generate an issue log document and send it to a list of project contacts, or you can leverage the project issue information to create an RFI or Pending Change Request.

1. From the Vista main men, select Project Management > Programs > PM Issues

1. From the PM Issues form, click the New Record () icon.

1. In the Project field, enter the project or press F4 to select from a list of projects.

1. In the Issue field, enter N, or '+' . . The system create a new issue and automatically assigns the next available sequential number.

1. The Project and Issue fields will populate with default information.

1. In the Description field, enter a brief description of the issue.

1. In the Type field, enter the type of project issue you are creating or press F4 to select from a list of issue types.

1. In the Details text box, enter detailed information about the issue.

Spelling Check

Click the Spelling icon on the toolbar or select Tools> Spelling to spell check the text in this field.

Add a Standard Note

Standard notes allow you to insert frequently used text into some fields in the application. This text is created and maintained using the HQ Standard Note form.

To insert a standard note into the field, right click the mouse while focus is in the field and select Standard Notes from the shortcut menu, which opens the Standard Note Copy window. Then enter the standard note to copy (or select from F4 lookup) and click OK. The system inserts the selected note into the field.Note: You can use the Add Standard Note feature to insert frequently used text into the issue details. For more information, see the [F1](/en/vista/vista/project-management/issuessubmittalsother-documents/project-issues/pm-issues-form/field-definitions-pm-issues-form#ID-00028d91--en) help.

1. Optional: Use the Distribution tab and Create and Send feature to generate an issue document and send it to a project contact.

1. Optional: Use the Issue Log Entry tab to create and maintain a list of actions associated with the issue.

1. Click Save when complete.
