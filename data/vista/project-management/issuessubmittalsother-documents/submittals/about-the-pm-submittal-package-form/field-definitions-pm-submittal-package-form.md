# Field Definitions: PM Submittal Package Form

The following is a list of field descriptions for the PM
 Submittal Package form. Many of the descriptions include links to other topics that provide additional information about or related to the topic.

## Project

Enter a project number or press F4 to select a
 project from a list.
[](/en/vista/vista/project-management/issuessubmittalsother-documents/submittals/about-submittals)PM Submittals - Overview
[](/en/vista/vista/project-management/issuessubmittalsother-documents/submittals/about-the-pm-submittal-package-form)PM Submittal Package

## Package

This field displays the submittal package code. Submittal packages allow you to group submittals together.
Create a new submittal package
Enter a new submittal package code. The submittal package code can be a combination of letters and/or numbers up to 10 characters long.
You can also enter a '+', 'n', or 'N' in this field and the system will automatically assign the package the next available number. This feature will not work if you have letters and/or special characters in the package number.
Select an existing package
Enter an existing submittal package number or
 press F4 to select a package from a list.
[](/en/vista/vista/project-management/issuessubmittalsother-documents/submittals/about-submittals)PM Submittals - Overview
[](/en/vista/vista/project-management/issuessubmittalsother-documents/submittals/about-the-pm-submittal-package-form)PM Submittal Package

## Revision

This field displays the revision code of the submittal package and it allows you to have multiple revisions of a single submittal package. The revision code can be up to five characters long, and it can include numbers and/or letters.
Select an existing revision
Enter the revision number or press F4 to select it
 from a list.
Create a new revision - Copy an existing revision
Click the Create Package Revision button at the
 bottom of the form to create a new submittal package revision. This allows you to copy the
 submittals on an existing revision to a new revision. [More](/en/vista/vista/project-management/issuessubmittalsother-documents/submittals/create-a-new-submittal-package-revision)
Create a new revision - Create a revision from scratch
Enter a revision number that does not exist yet or enter a '+' and the system will automatically assign the revision the next available number (this only works if the revision code does not include letters).
This field will default to 0 when you are creating a new submittal package.

## Status

Enter the status of the submittal package or
 press F4 to select a status code from a list.
When you change the status of a submittal package, you can also do the following:

- Assign to all submittals - If you change the status of an existing
 submittal package, the system gives you the option of applying that status to all of the
 submittals on the package. Submittals that have already been assigned a final status
 will not be updated.

- Close the package - If you assign the submittal package a final
 status, the system automatically closes the submittal package and all of the submittals
 on the package.

Status codes are created and maintained using
 the [PM
 Status IDs](/en/vista/vista/project-management/setupmaintenance/pm-status-ids-form) form. You can open this form by pressing F5 in this
 field.
Click [here](/en/vista/vista/project-management/issuessubmittalsother-documents/submittals/about-submittals) for an overview on statuses, and opening/closing submittals and submittal packages.

## Activity ID

You can use this field to store third-party
 reference numbers. An activity ID can be added to the submittal package, and to each
 submittal revision using the Activity ID field on the [PM
 Submittal Register](/en/vista/vista/project-management/issuessubmittalsother-documents/submittals/about-the-pm-submittal-register-form) form.
Enter an activity ID or press F4 to select
 one from a list. There are several options on the lookup form to help you locate the
 correct activity ID.
Note: The activity IDs entered on submittal packages are not
 stored in the same place as the activity IDs entered on submittals. This means that when
 you press F4, the activity IDs that you have entered on submittals using the PM
 Submittal Register form will not display in the list.

To insert a standard note into the field,

 note to copy (or select from F4 lookup) and click OK. The system inserts the selected note

## Sequence

The sequence identifies each unique submittal
 revision. This field also displays on the PM Submittal Register form (Sequence
 field).
You can use this field to:

- Select a submittal that has been entered on the register ([PM Submittal Register](/en/vista/vista/project-management/issuessubmittalsother-documents/submittals/about-the-pm-submittal-register-form)). This will add the submittal to the package.

- Create a new submittal revision. This will create a new submittal revision on the package and a new entry on the submittal register. Generally, it's easier to create submittals using the register (PM Submittal Register) and then add them to a submittal package using this form.
Add multiple submittals already entered on the register

Click the Add Submittals button at the bottom of
 the form to add multiple submittals on a register to the package. [More](/en/vista/vista/project-management/issuessubmittalsother-documents/submittals/add-submittals-to-a-submittal-package)

There are several ways to select a submittal that has already been created using the [PM Submittal Register](/en/vista/vista/project-management/issuessubmittalsother-documents/submittals/about-the-pm-submittal-register-form) form.

- Click the Add Submittals button to add a
 submittal that has already been entered on a register using the [PM Submittal Register](/en/vista/vista/project-management/issuessubmittalsother-documents/submittals/about-the-pm-submittal-register-form) form.  record in the grid, and select a submittal from a list.

 Submittal

 the Grid tab that have an approaching due date. For example enter a "2" in this field and

 Firm, Due from Responsible Firm, Due to Approving

 form.

## Notes

Use this field to enter notes on a specific submittal in the package.

Spelling Check
 Click the Spelling icon on the toolbar or
 select Tools >  Spelling
  to spell check the text in this field.
Add a Standard Note
Standard notes allow you to insert frequently used text into some fields in the application. This text is created and maintained using the [HQ Standard Note](/en/vista/vista/administration/headquarters/standard-notes/about-the-hq-standard-note-form) form.
To insert a standard note into the field,
 right click the mouse while focus is in the field and select Standard Notes
 from the shortcut menu, which opens the Standard Note Copy window. Then enter the standard
 note to copy (or select from F4 lookup) and click OK. The system inserts the selected note
 into the field.
