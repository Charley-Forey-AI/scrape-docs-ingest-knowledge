# Field Definitions: PM Submittal Register Form

The following is a list of field descriptions for the PM
 Submittal Register form. Many of the descriptions include links to other topics that provide additional information about or related to the topic.

## Project

 Enter a project number or press F4 to select a
 project from a list.

[](/en/vista/vista/project-management/issuessubmittalsother-documents/submittals/about-the-pm-submittal-register-form)PM Submittal Register

## Display submittals due in

Use this field to highlight the submittals that have an approaching due date. For example enter a "2" in this field and any submittal with a Due to
 Responsible Firmdate within the next 2 days and no Sent to Responsible
 Firm date will be highlighted.
This feature applies to the Due to Responsible
 Firm, Due from Responsible Firm, Due to Approving
 Firm, Due from Approving Firm fields on this form.
You can leave this field blank if you do not want to use this feature.
There is a similar field in the lower portion of the [PM Submittal Package](/en/vista/vista/project-management/issuessubmittalsother-documents/submittals/about-the-pm-submittal-package-form) form.
Note: The selection in this field is saved in your user account - for example if you enter a "5" in this field and then close the form, this field will display a "5" the next time that you open the form.

## Display most current revision

Check this box if only the most recent revision of each submittal should display in the form. For example if a submittal has multiple revisions, only the submittal with the highest revision number will display.
Note:Generally this filter will work even if your revision numbers include letters, dashes(1-4), or periods (1.1). This filter may not work if your revision numbers include special characters.

The selection in this box is saved in your user account - for example if you check this box and then close the form, this box will be checked the next time that you open the form.

## Submittal Number

The submittal number identifies each unique submittal. It can be up to 10 characters long and it can include numbers and/or letters.

## Submittal Revision

 Use this field to enter the revision number of the submittal. The revision number must be unique.

- Copy Existing- Click the Create Revision button at the bottom

## Activity ID

You can use this field to store third-party
 reference numbers. An activity ID can also be added to submittal packages using the
 Activity
 ID field on the [PM
 Submittal Package](/en/vista/vista/project-management/issuessubmittalsother-documents/submittals/about-the-pm-submittal-package-form) form.
 Enter an activity ID or press F4 to select
 one from a list. All Activity IDs that have already been entered using this form will
 display in the list.
Note: The activity IDs entered on submittals are not stored
 in the same place as the activity IDs entered on submittal packages. This means that when
 you press F4, the activity IDs that you have entered on submittal packages will not
 display in the list.

Spelling Check
 Click the Spelling icon on the toolbar or
 select Tools >  Spelling
  to spell check the text in this field.
Add a Standard Note
Standard notes allow you to insert frequently used text into some fields in the application. This text is created and maintained using the [HQ Standard Note](/en/vista/vista/administration/headquarters/standard-notes/about-the-hq-standard-note-form) form.
To insert a standard note into the field,
 right click the mouse while focus is in the field and select Standard Notes
 from the shortcut menu, which opens the Standard Note Copy window. Then enter the standard
 note to copy (or select from F4 lookup) and click OK. The system inserts the selected note
 into the field.
