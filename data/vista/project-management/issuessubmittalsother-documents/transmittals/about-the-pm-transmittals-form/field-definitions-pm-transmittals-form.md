# Field Definitions: PM Transmittals Form

The following is a list of field descriptions for the PM
 Transmittals form. Many of the descriptions include links to other topics that provide additional information about or related to the topic.

##  Project

 Enter the project for which to add/edit a transmittal.
[](/en/vista/vista/project-management/issuessubmittalsother-documents/transmittals/about-the-pm-transmittals-form)PM Transmittals

## Transmittal

Create a transmittal
There are a few ways to create a new transmittal using this field:

- Enter a '+' and then press TAB. The system will create a new transmittal and automatically assign it the next available number.

- Click the New Record icon () at the top of the form. The system will automatically create a new transmittal and assign it the next available transmittal number.

Note: The system will ignore all alphanumeric numbers when selecting the next available number.
Select an existing transmittal

 Enter an existing transmittal number or press F4 to select one from a list.
[](/en/vista/vista/project-management/issuessubmittalsother-documents/transmittals/about-the-pm-transmittals-form)PM Transmittals

## Subject

Enter the subject of the transmittal. The subject can be up to 255 characters long.
[](/en/vista/vista/project-management/issuessubmittalsother-documents/transmittals/about-the-pm-transmittals-form)PM Transmittals

##  Document Type

 Use this field to categorize the transmittal
 that you are creating. Press F4 to select a document type from a
 list.
Document types are created and maintained
 using the [PM Document Types ](/en/vista/vista/project-management/setupmaintenance/pm-document-types-form) form. Click [here](/en/vista/vista/project-management/setupmaintenance/pm-document-types-form) for more information on document types.

##  Responded Date

 Specify the actual date you received a response for this transmittal. This date can be used by Notifier to identify transmittals for which you have not received a response.
[](/en/vista/vista/project-management/issuessubmittalsother-documents/transmittals/about-the-pm-transmittals-form)PM Transmittals

## Responsible Person

Specify the person responsible for this transmittal. Must be a valid contact defined for the 'Project Our Firm' in PM Projects or, if no ‘our firm’ specified for the project, the ‘Our Firm’ specified in PM Company Parameters.
[](/en/vista/vista/project-management/issuessubmittalsother-documents/transmittals/about-the-pm-transmittals-form)PM Transmittals

## Sequence

The sequence number identifies a specific document on the Documents tab.
Add a new document to the transmittal
There are a few ways to add a new document to the transmittal:

- When the Documents tab is open, click the New Record
 icon () at the top of the form. This
 will automatically populate the Sequence field and create a new
 record on the Documents tab.

- Enter a '+' in this field and the system will
 automatically assign the record the next available number.

[](/en/vista/vista/project-management/issuessubmittalsother-documents/transmittals/about-the-pm-transmittals-form)PM Transmittals

##  Document Type

 Indicate the type of document (from PM Document Types) being attached this transmittal. May be left blank to allow setting up transmittals 'on the fly'.
 Documents added using PM Transmittal Add will default the document type from the source document specified when the transmittal was added.
[](/en/vista/vista/project-management/issuessubmittalsother-documents/transmittals/about-the-pm-transmittals-form)PM Transmittals

##  Document

 Enabled only if a Document Type was specified for this sequence.
 Specify the document to attach to this transmittal. Press F4 for a list of documents available for the specified Document Type. May be left blank to allow setting up transmittals 'on the fly'.
 Documents added using PM Transmittal Add will default the document number from the source document specified when the transmittal was added.
[](/en/vista/vista/project-management/issuessubmittalsother-documents/transmittals/about-the-pm-transmittals-form)PM Transmittals

##  Revision

 This field is only accessible for submittal documents (document types with a category of ‘Submittals’).
 Specify the revision of the selected submittal document to attach to this transmittal. Press F4 for a list of valid revisions for the selected submittal.
Note:Documents added using PM Transmittal Add will default the revision number from the source document specified when the transmittal was added.

[](/en/vista/vista/project-management/issuessubmittalsother-documents/transmittals/about-the-pm-transmittals-form)PM Transmittals

## Drawing Rev

This field is only accessible for drawing logs (document types with a category of ‘Drawing Logs’).
Specify the revision of the selected drawing log to attach to this document. Press F4 for a list of valid revisions for the selected drawing log.
Note:Documents added using PM Transmittal Add will default the revision number from the source document specified when adding the transmittal.

[](/en/vista/vista/project-management/issuessubmittalsother-documents/transmittals/about-the-pm-transmittals-form)PM Transmittals

## Submittal / Package Rev

Use this field to select the revision number
 of the submittal or submittal package selected in the Document  field. Press F4 to select a
 revision number from a list.
This field is only enabled when a submittal
 package or submittal register document type is selected in the Document Type
 field.
[](/en/vista/vista/project-management/issuessubmittalsother-documents/transmittals/about-the-pm-transmittals-form)PM Transmittals

## Document Description

Enter a description for this document, up to 60 characters. Defaults the description from the source document. This is the description that will print on the transmittal for this document.
[](/en/vista/vista/project-management/issuessubmittalsother-documents/transmittals/about-the-pm-transmittals-form)PM Transmittals

##  Copies Sent

 Specify the number of copies (0-255) of this document that were sent out.
[](/en/vista/vista/project-management/issuessubmittalsother-documents/transmittals/about-the-pm-transmittals-form)PM Transmittals

## Status

Enter the status of this transmittal or press F4 to select it from a list.
Statuses are created and maintained using the [PM Status IDs](/en/vista/vista/project-management/setupmaintenance/pm-status-ids-form) form.
[](/en/vista/vista/project-management/issuessubmittalsother-documents/transmittals/about-the-pm-transmittals-form)PM Transmittals

## Remarks

Enter any remarks or notes regarding this
 transmittal. For additional space in which to write more notes, double-click on this field.
 The Grid Notes window will display, allowing you virtually unlimited space for your notes
 and information. There is however, a maximum allowance of 8k.
Spelling Check
 Click the Spelling icon on the toolbar or
 select Tools >  Spelling
  to spell check the text in this field.
Add a Standard Note
Standard notes allow you to insert frequently used text into some fields in the application. This text is created and maintained using the [HQ Standard Note](/en/vista/vista/administration/headquarters/standard-notes/about-the-hq-standard-note-form) form.
To insert a standard note into the field,
 right click the mouse while focus is in the field and select Standard Notes
 from the shortcut menu, which opens the Standard Note Copy window. Then enter the standard
 note to copy (or select from F4 lookup) and click OK. The system inserts the selected note
 into the field.
[](/en/vista/vista/project-management/issuessubmittalsother-documents/transmittals/about-the-pm-transmittals-form)PM Transmittals

## Notes

To insert a standard note into the field,
 right click the mouse while focus is in the field and select Standard Notes
 from the shortcut menu, which opens the Standard Note Copy window. Then enter the standard
 note to copy (or select from F4 lookup) and click OK. The system inserts the selected note
 into the field.
Spelling Check
 Click the Spelling icon on the toolbar or select Tools >  Spelling  to spell check the text in this field.
