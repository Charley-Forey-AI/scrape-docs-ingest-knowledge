# Field Definitions: PM Firm Contacts Form

The following is a list of field descriptions for the PM Firm
 Contacts form. Many of the descriptions include links to other topics that provide additional information about or related to the topic.

## Firm Number

Enter the firm number or press F4 to select a
 firm from a list.
Tip:The sort name can be used instead of the firm number
 when selecting firms in PM module forms. Generally the sort name is the first several
 letters of the firm name.  For example when creating a subcontract in the [PM Subcontracts](/en/vista/vista/project-management/subcontracts/pm-subcontracts-form) form for 'Bryan Electrical', you can
 enter the sort name 'bryan' in the Vendor field instead of the firm
 number '10042'. The sort name of a firm is set up using the Sort Name
 field on the [PM Firms](/en/vista/vista/project-management/setupmaintenance/about-the-pm-firms-form) form.

Note: If you accessed this form via the PM Firms (Contacts
 tab), this field defaults the active firm and is disabled.

## Contact Code

Create a new contact
When you create a new contact, you are
 creating a contact at the firm selected in the Firm Number field.
There are several ways to create a new contact:

- Enter a "+" in this field to create a new contact at
 the firm selected in the Firm Number field, and then press
 TAB. The system will automatically assign the new contact the next available contact
 code.

- Click the New Record icon (), and then select a firm in the
 Firm
 Number field. The Contact Code field will
 automatically populate with the next available code for the selected firm. Press TAB
 to exit the field and accept the default.

- Enter a code that has not been used on the firm
 selected in the Firm Number field. The code number
 can be up to 6 digits long.

Note:If you used the Initialize feature to generate contacts for this firm using the [PR Employees ](/en/vista/vista/hr-and-payroll/payroll/setupmaintenance/employee/pr-employees-form) form, this field defaults a code identical to the employee number.
Select an existing contact

 Enter an existing contact code or press
 F4
 to select a contact from a list.

## Preferred Method of Correspondence

Select the contact’s preferred method of correspondence. The method selected here will determine how project documents are distributed to this person.

- Print – Select this option if this person prefers to receive all correspondence in hard copy (printed).

- Email – Select this option if this person prefers to receive all correspondence by email. Make sure to enter the email address in the space provided to the right of this field.

- Fax – Select this option if this person prefers to receive all correspondence via fax. Make sure you enter the fax number in the space provided to the right of this field.
Create and Send

The Create and Send feature is used to generate project documents and/or project correspondence. The selection in this field determines by default what kind of correspondence is created for the contact when the Create and Send feature is used. This is only the default.
Click [here](/en/vista/vista/project-management/create-and-send/about-the-create-and-send-feature) for an overview on the Create and Send feature.

## Formatted Fax

 Use this field to enter a custom formatted
 fax number. When there is a value in this field, it will be used instead of the value in
 the Fax
 field when the preferred contact method is Fax, and the value in the Fax Server Name
 field on the [PM Company Parameters ](/en/vista/vista/project-management/setupmaintenance/pm-company-parameters-form) form will not be appended to it.
This field allows you to enter custom formatted fax numbers that do not meet the standard <##########@faxservername> format. For example if you use the RightFax Fax Server, you can use this field to enter fax numbers using the format: <FirstName@##########>.
Create and Send

 Name field). [More](/en/vista/vista/project-management/setupmaintenance/pm-company-parameters-form/field-definitions-pm-company-parameters-form#ID-00023838--en)

##  Use Contact Address

 Check this box to use this contact’s mailing address instead of the firm’s mailing address. Checking this option enabled the contact address fields below.
The system will only use this address if the contact is included on a document’s distribution list and has a CC method of ‘None’.)
 Leave this box unchecked to use the firm’s mailing address information. Unchecking this box disables the contact address fields below; however, any address information previously entered for the contact remains intact.
Note:In order to use the contact’s address on documents, you will need to add the override address fields (OvrdMailAddress, OvrdMailAddress2, OvrdMailCity, OvrdMailCountry, OvrdMailState, and OvrdMailZip)) to applicable templates in PM Create & Send Templates (Merge Fields tab). In addition, to include this address in PM Document Tracking, you must add the address fields to the document tracking view for each applicable grid form in PM Document Tracking Grids.

## Mail Country

Enter the 2-character country code for this contact. Entry in this field is required when the address exists outside the Default Country specified in HQ Company Setup for the active company. Country must be valid for the specified state (e.g. state, province, territory, etc.) as defined in HQ States.
If you did not specify a country, the system
 uses the Default
 Country defined in HQ Company Setup.
