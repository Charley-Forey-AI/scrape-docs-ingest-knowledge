# Field Definitions: PM Create and Send Locations Form

The following is a list of field descriptions for the PM
 Create and Send Locations form.Many of the descriptions include links to other topics that provide additional information about or related to the topic.

## Location

Enter a document location, up to 10
 characters.
If you see a red-colored READ ONLY label, the form and its assigned locations are read-only
 and you are not able to change any field values. You can change this setting in the
 Viewpoint Configuration form on the server.
Note:The standard document templates provided by Viewpoint (as a starting point for creating your own templates) use a ‘PMStandard’ location; however, user-created document templates cannot be assigned this location, so you will need to set up at least one location to use with your document templates.

## Document Path

Note: Your company's deployment method affects which actions you should take on this page.
 Your deployment method is shown at the bottom of the Main Menu in the status bar.If your
 deployment method is VRL Cloud or VRL on-premises, this field does not appear and the
 information on this page does not apply to you.

Enter the path name of your document templates or use the Browse button to locate the directory in which your documents are stored. Once you have located the correct location, click the OK button (in the Browse for Folder window) and the location will be updated to the Document Path field.
Note:It is strongly recommended that you do not use the
 standard Viewpoint document templates directory (Viewpoint Repository\Document
 Templates\Standard). When you install a release, the update overwrites this directory,
 thereby removing your custom templates.
Instead, use the custom directory
 (Viewpoint Repository\Document Templates\Custom).
