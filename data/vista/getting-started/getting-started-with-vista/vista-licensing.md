# Vista Licensing

Vista has different license models and types, which you can view and track in several different ways.

## License Models

Your organization uses either a NUL or CUL license model.NUL: Named User License

- Only cloud deployments can use the NUL
 license model.

- NUL enforcement occurs when attempting to
 set up a new user in VA User Profile.

- NUL models provide the option to choose
 license types to limit users from accessing certain forms or
 reports in Vista. The two license types are
 Office or PM.
 For more information, see [License Types](/en/vista/vista/getting-started/getting-started-with-vista/vista-licensing/license-types)

CUL: Concurrent User License

- On-premises deployments use the CUL license
 model. (Some cloud deployments use CUL.)

- CUL enforcement occurs at login, as it is
 based on the maximum number of users who are currently signed
 in.

## Licensing Options

There are a variety of options for allocating and auditing licenses depending on your
 Vista cloud or on-premises deployment method and NUL or CUL license model.
On-demand licensingOn-demand licensing is an option for
 cloud-deployed organizations using either NUL or CUL model. For more
 information, see [Set On-Demand Licensing Usage](/en/vista/vista/getting-started/getting-started-with-vista/vista-licensing/set-on-demand-licensing-usage).
Track license usage and update license countsFor information about various ways you can track
 licenses and identify which forms and reports require specific
 license types, see [Track License Counts and Usage](/en/vista/vista/getting-started/getting-started-with-vista/vista-licensing/track-license-counts-and-usage).
For information about reviewing named license
 counts, see [Audit and Update Licenses in Vista](/en/vista/vista/getting-started/getting-started-with-vista/vista-licensing/audit-and-update-licenses-in-vista).
