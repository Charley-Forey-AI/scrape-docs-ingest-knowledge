# Account Creation

Depending on your Vista deployment method, organizational
 setup, and user needs, there are several different accounts you could create in Vista. You
 set up all Vista accounts in VA User Profile.

## Vista User Accounts

The following accounts are for users who require access to the
 Vista application.

- Vista User
 SSO Account with Trimble ID: For Trimble Construction One
 cloud-hosted customers, you should set up new Vista users with an SSO
 account that uses Trimble ID login credentials. For details, see [Create a Vista User SSO Account with Trimble ID](/en/vista/vista/getting-started/getting-started-with-vista/account-creation/create-a-vista-user-sso-account-with-trimble-id).

- Vista User
 SQL Classic Login Account: For legacy cloud and on-premises
 customers, set up your Vista users with SQL credentials that they can use to
 log in to Vista. For details, see [Create a Vista SQL User
 in VA User Profile](/en/vista/vista/getting-started/getting-started-with-vista/account-creation/create-a-vista-sql-user-in-va-user-profile).

- Vista User
 Windows Domain Login Account: For legacy cloud and
 on-premises customers using Azure Active Directory (now part of Microsoft
 Entra ID), set up your Vista users with the same Windows credentials they
 use to access their workstation. For details, see [Create a Vista Domain User](/en/vista/vista/getting-started/getting-started-with-vista/account-creation/create-a-vista-domain-user).

## Non-User Accounts

The following accounts do *not*
 grant access to the Vista application. These accounts are intended for specific
 scenarios.

- Vista
 Database Account: For service accounts, integrations,
 third-party applications, SQL logins, and database logins that do *not* require logging in to the Vista
 application, you can create a database account. This account will have
 access to the Vista database but will *not*
 have access to the Vista application. For details, see [Create a Database Account for Non-Vista Users](/en/vista/vista/getting-started/getting-started-with-vista/account-creation/create-a-database-account-for-non-vista-users).

- Trimble
 Construction One Analytics Account: For Trimble Construction
 One Analytics users who do *not* require
 logging in to the Vista application but who *do* require linking Vista security to a Trimble ID, you can create a
 non-Vista user Trimble ID account. This type of account is useful if you
 access products such as Trimble Construction One Analytics or specific
 workflows in Vista Web. This account will not have access to the Vista
 application. For details, see [Create a Trimble Construction One Analytics Account for Non-Vista Users](/en/vista/vista/getting-started/getting-started-with-vista/account-creation/create-a-trimble-construction-one-analytics-account-for-non-vista-users).
