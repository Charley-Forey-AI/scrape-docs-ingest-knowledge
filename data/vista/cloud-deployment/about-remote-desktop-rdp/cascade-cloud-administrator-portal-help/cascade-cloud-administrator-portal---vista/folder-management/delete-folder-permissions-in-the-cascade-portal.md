# Delete Folder Permissions in the Cascade Portal

You can remove folder permissions for users or
 groups.
Note:Each folder
 must have at least one user or group with permissions. If there is only one user
 or group for a particular folder, the delete icon will be disabled  (grayed out) and you will not be able to remove
 permissions for this user or group.

1. Select
 the folder icon  to open the Folder
 Management page.

1. For the folder you want to remove permissions
 from, select the edit icon  to
 open the Permissions Management page.

1. In the Current Permissions section, for the user
 or group you want to remove folder permissions from, select the delete icon
 .You will see the changed permissions labeled as Delete Pending.

1. To confirm the delete, select Save Changes.Any permissions for the user or group will be removed from
 that particular folder.

To add new user or group permissions, see [Add Folder Permissions in the Cascade Portal](/en/vista/vista/cloud-deployment/about-remote-desktop-rdp/cascade-cloud-administrator-portal-help/cascade-cloud-administrator-portal---vista/folder-management/add-folder-permissions-in-the-cascade-portal).
