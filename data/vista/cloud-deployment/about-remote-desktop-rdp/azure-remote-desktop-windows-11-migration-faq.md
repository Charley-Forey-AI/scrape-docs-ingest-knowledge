# Azure Remote Desktop Windows 11 Migration FAQ

Learn about the upcoming Azure Remote Desktop Windows 11 migration. These migrations will impact cloud customers using Remote Desktop Protocol (RDP) to access Trimble applications and occur starting in November 2024, going through June 2026.
This Windows 11 migration effort impacts Vista, ProContractor, and Viewpoint Estimating RDP customers.

## What is changing?

We are updating the back-end operating system of our Azure Remote Desktop client to Windows 11. This involves replacing the current server with a new one.

- You will not see any changes to the front-end, as we are only updating the back-end operating system.

- Your user profile data will be migrated to a more robust and modern storage infrastructure.

- For Vista customers, your Vista, Microsoft Office, and Chrome applications will transition to a 64-bit architecture. For more details, see the [64-bit Upgrade FAQ](https://apim.viewpointplatform.com/help/v1/contents/urn:contentId:vista_60054).

## Who is impacted?

All cloud customers using the Remote Desktop Protocol (RDP) services to access Trimble applications will be upgraded to Windows 11. RDP services will be unavailable while the migration takes place, which will occur after standard business hours and last up to 60 minutes.
Note: If you are a Vista customer accessing Vista through VRL, this migration to Windows 11 does *not* impact you.

## When will the migration occur for my organization?

Migrations will take place between November 2024 through the end of 2025. At least one week before your scheduled migration date, you will receive an email notification with more information.
The migration will occur outside of standard business hours.

## Why is this change taking place?

This migration is part of a required maintenance optimization effort to consolidate licensing costs and modernize our architecture. This will ultimately enhance your Remote Desktop experience.

## Is this required?

Yes, all RDP customers must be migrated to Windows 11. While you cannot opt out of this update, you can reschedule. To change your migration date, please contact windows11migration-ug@trimble.com.

## How long will the migration take?

Most of the migration preparations will be completed the day before the migration. On the migration day, the process will take approximately 30 to 60 minutes.

## What actions do I need to take?

If you use third-party applications, you may need to take some additional steps.
We will be sending additional communications to customers with a prior agreement to host third-party software. You may need to provide license keys and coordinate with your third-party vendors.

## Will I be able to use my application during the migration?

No, however, the migration will take place outside of standard business hours and only last about 60 minutes.

## What happens if there are issues with the migration?

If needed, our engineers have a rollback plan to revert to the original server. These original servers will remain operational for 30 days after the migration.

## Are there any training resources available?

Training will not be needed, as this migration only involves back-end changes. You will not see any changes to the user interface or user experience.
