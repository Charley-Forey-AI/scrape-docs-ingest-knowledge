# Copy Local Files to the Cloud

If you use Vista in Trimble Viewpoint's cloud, you must take certain actions in order to attach any files from your local workstation to Vista records.
Drag and drop works between Vista and the other applications within the RDP environment, but not directly between your RDP-deployed Vista application and your local applications.To copy files from your local workstation into the cloud:

1. Open the cloud File Explorer.An icon with green arrows appears in your toolbar, confirming you are working in the cloud File Explorer.

1. Locate the local file(s) you want to copy into the cloud. Options include but are not limited to:

- email attachments

- documents on your desktop or file explorer

- files on a network drive

1. Copy and paste the file(s) to the cloud File Explorer.

The file(s) can now be attached to Vista records as needed.If you want to copy one or more files from the cloud File Explorer to your local workstation, the same process applies.

This brief video shows how to copy and paste to and from the cloud File Explorer.Note: You must use a browser on your local computer to view this video, and not a browser in the cloud. If you are currently using a cloud-based browser, copy this page's URL and paste it into a web browser on your local workstation.
