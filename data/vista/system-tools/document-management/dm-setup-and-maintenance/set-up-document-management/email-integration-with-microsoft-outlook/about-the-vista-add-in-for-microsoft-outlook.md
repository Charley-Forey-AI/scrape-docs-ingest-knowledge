# About the Vista Add-in for Microsoft Outlook

The Outlook Add-in enables you to use Outlook to index emails to Vista records.
During the process of installing the Vista client on your workstation, the installation program asks whether or not you want to install the Viewpoint add-in for MS Outlook. If you choose to install it, the installer program installs it for you.
Note:

- the user login must have administrative rights to the workstation and must have previously configured Outlook. If your login does not have administrative rights, contact your system administrator.

- Outlook must be installed and functioning for the workstation user prior to installing Vista.

- the user login that will be using the Viewpoint Add-in must be the login to install it.

- in cases of Citrix/Terminal Servers, the user permissions must be elevated for the period of time it takes to install the Add-in. Once complete, you can lower the permissions to normal.

The only way to get the add-in installed on your workstation is by using the Vista client installer. So if you choose not to install the Viewpoint add-in, and decide at a later date to install it, you must run the Vista client installer again on your workstation, making sure not to disable the Outlook Add-In from the Custom Setup installer screen. If needed, see [Install the Vista Client on a Workstation](/en/vista/vista/on-premises-deployments/installations/major-release-installation/vista-client-application/install-the-vista-client-on-a-workstation).
If you do not install the add-in, you will not be able to send emails directly from Outlook to Vista. Instead, you will only be able to attach emails to data records by dragging and dropping the email onto a data record.If you wish to disable the installed Viewpoint Add-in, do so in Microsoft Outlook (File > Options > Add-Ins).
