# Setting Report Types

You can set up report types, customize existing standard report types, and reset customized reports back to standard status.
 Report types are assigned to reports to classify the different types of reports used by your company. They can also be used as a filter when assigning security in [VA Report Security](/en/vista/vista/administration/viewpoint-administration/application-security/data-record-and-content-security/va-report-security-form).
Some examples of report types are:

- Accounting

- Project Management

- Audit

- Forms (for pre-printed forms)

- Drill Down

- Error

To set a report type:

1. In RP Report Type, in
 the Report Type field, enter a code that identifies this type of
 report. If you want to edit an exiting report type or customize a standard
 report type, you can also press F4 to select from a list of
 existing report types.

1. In the Description field, enter a description of the report
 type.

1. Clear the Active checkbox if you do not want to activate this report
 type. Activating the report type makes it available in RP Report Titles. All
 standard report types are labeled as "Standard". Custom report types or
 overridden standard types are labeled as “Custom”.Now you can assign your report
 types to reports in RP Report Titles.
To reset a customized standard
 report back to its standard status:

1. In RP Report Type, in
 the Report Type field, F4 to select the customized
 report type.

1. Click Reset. The report type is returned to a standard status.
