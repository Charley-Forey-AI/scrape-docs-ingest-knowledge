# WF Checklist Template

Use this form to create checklist templates. Templates are useful
 when you require a number of similar checklists, each with minor differences. For example,
 you might create an AP processing checklist and create a number of instances based on the
 template (e.g., June AP Processing, July AP Processing, etc.). Then you can use [WF Checklist
 Initialization](/en/vista/vista/system-tools/work-flow/checklists/wf-checklist-initialization) to create specific checklists based on a template.
Checklist templates are available for all companies set up in the
 system. However, when you create a checklist, the system ties it directly to your current
 company and does not cross company boundaries.
This form comes with standard implementation and processing
 checklists. You cannot delete these standard checklists, but you can use them to create
 your own checklists. If the checklist is a standard checklist, the system marks the
 Standard checkbox. The Standard checkbox is inaccessible and you cannot change its
 setting.
The form displays the template creator and the creation date in
 the Added By and Date fields. Additionally, the form stores the last modification date for
 the template, as well as the user who modified the template in the Changed By and Date
 fields. The Changed By and Date fields change every time you make a change to the template
 and save the record.
The Revised field automatically defaults to the current date when
 you create a template. This field does not change when you save a template record and
 retains the original creation date until changed. You can use this field to keep track of
 checklist versions/revisions. For example, you may not want to change the revision date
 when fixing minor issues to a checklist, but it might be necessary when you add a task or
 step.
