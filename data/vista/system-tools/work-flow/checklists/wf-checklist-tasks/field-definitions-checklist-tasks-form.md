# Field Definitions: Checklist Tasks Form

The following is a list of field descriptions for the WF
 Checklist Tasks form. Many of the descriptions include links to other topics that provide additional information about or related to the topic.

##  Checklist

 Enter the checklist to review. Press F4 for a list of checklists.

## Checklist Item Status Type

Select one, or multiple status types for filtering. If you deselect all checkboxes, the system automatically reselects the New and In Progress boxes for the search.

- New – Use this option to search for all task and steps with a new status.

- In Progress – Use this option to search for all tasks and steps with an ‘in progress’ status.

- Final – Use this option to search for all completed tasks and steps.

## Search By Date

Select a date filtering option.

- None – Select this option if you do not want to use a date filtering option.

- Due – Select this option to search for tasks and steps based on the due date.

- Assigned – Select this option to search for tasks and steps based on the assigned date.

If you select either Due or
 Assigned, specify a date range in the Beginning Date and Ending Date
 fields.

## Beginning Date/Ending Date

If you select either the Due or
 Assigned radio button in the Search By Date section, specify a date
 range in these fields.
Note: If you do not enter a date in either field, the system
 automatically enters yesterday’s date in the Beginning Date field and a date
 fifteen days later in the Ending Date field.

## Summary

This field displays a brief summary of the task. You can override this field as necessary, up to 256 characters.
Note: Changes to this field update the task information for the checklist. The system reflects the change on the WF Users Tasks and WF Checklist Maintenance forms.

## Required

This checkbox displays whether the task is required or not. You can override this field as necessary.
Note: Changes to this field update the task information for the checklist. The system reflects the change on the WF Users Tasks and WF Checklist Maintenance forms.

## Use Notification

This checkbox displays whether or not the system notifies task assignees when the task is ready. You can override this field as necessary.
Note: Changes to this field update the task information for the checklist. The system reflects the change on the WF Users Tasks and WF Checklist Maintenance forms.

## Status

This field displays the status of the task. You can override this field as necessary.
Note: Changes to this field update the task information for the checklist. The system reflects the change on the WF Users Tasks and WF Checklist Maintenance forms.
If you see these codes: 999997, 999998, or 999999, you should not use them. Instead, set up status codes for New, In Progress, and Final in WF Checklist Status Codes.

## Description

This field displays the task description. You can override this field as necessary, up to 2,000 characters.
Note: Changes to this field update the task information for the checklist. The system reflects the change on the WF Users Tasks and WF Checklist Maintenance forms.

## Task Type

This section displays the task type. You can
 override the selection as necessary. If you change the type to either Form or
 Report, the system enables the Form/Report Name field.
Note: Changes to this field update the task information for the checklist. The system reflects the change on the WF Users Tasks and WF Checklist Maintenance forms.

## Form/Report Name

This field displays the form or report associated with this task. This field is accessible only when either ‘Form’ or ‘Report’ is the task type. You can override this field as necessary.
Note: Changes to this field update the task information for the checklist. The system reflects the change on the WF Users Tasks and WF Checklist Maintenance forms.

## Assigned To

This field displays the user assigned to the task. You can override this field as necessary.
Note: Changes to this field update the task information for the checklist. The system reflects the change on the WF Users Tasks and WF Checklist Maintenance forms.

## Assigned Date

This field displays the assigned date for the task. You can override this field as necessary.
Note: Changes to this field updates the task information for the checklist. The system reflects the change on the WF Users Tasks and WF Checklist Maintenance forms.

## Due Date

This field displays the due date for the task. You can override this field as necessary.
Note: Changes to this field updates the task information for the checklist. The system reflects the change on the WF Users Tasks and WF Checklist Maintenance forms.

## Completed By

This field displays the user who completed the task. You can override this field as necessary or enter a user name if one does not exist.
Note: Changes to this field updates the task information for the checklist. The system reflects the change on the WF Users Tasks and WF Checklist Maintenance forms.

## Completed Date

This field displays the completion date for the task. You can override this field as necessary or enter a date if one does not exist.
Note: Changes to this field updates the task information for the checklist. The system reflects the change on the WF Users Tasks and WF Checklist Maintenance forms.

## Step

This field displays the step number. The system does not allow overrides to this field. To change the step order, use the WF Checklist Maintenance form.
