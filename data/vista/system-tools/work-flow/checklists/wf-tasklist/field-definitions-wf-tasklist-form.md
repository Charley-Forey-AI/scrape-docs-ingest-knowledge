# Field Definitions: WF Tasklist Form

The following is a list of field descriptions for the WF
 Tasklist form. Many of the descriptions include links to other topics that provide additional information about or related to the topic.

## Checklist Item Status Type

Select one, or multiple status types for filtering. If you deselect all checkboxes, the system automatically reselects the New and In Progress boxes for the search.

- New – Use this option to search for all task and steps with a new status.

- In Progress – Use this option to search for all tasks and steps with an ‘in progress’ status.

- Final – Use this option to search for all completed tasks and steps.

## Search By Date

Select a date filtering option.

- None – Select this option if you do not want to use
 a date filtering option.

- Due – Select this option to search for tasks and
 steps based on the due date.

- Assigned – Select this option to search for tasks
 and steps based on the assigned date.

If you select either Due or
 Assigned, specify a date range in the Beginning Date and Ending Date
 fields.

## Beginning Date/Ending Date

If you select either the Due or
 Assigned radio button in the Search By Date section, specify a date
 range in these fields.
Note: If you do not enter a date in either field, the system
 automatically enters yesterday’s date in the Beginning Date field and a date
 fifteen days later in the Ending Date field.

##  Status

 Enter a status for the task/step, or press
 F4
 to select from a list of valid statuses.
If you see these codes: 999997, 999998, or 999999, you should not use them. Instead, set up status codes for New, In Progress, and Final in WF Checklist Status Codes.

## Completed Date

 Enter the date you completed the task.
