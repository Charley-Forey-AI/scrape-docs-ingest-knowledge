# Field Definitions: WF Checklist Maintenance Form

The following is a list of field descriptions for the WF
 Checklist Maintenance form. Many of the descriptions include links to other topics that provide additional information about or related to the topic.

##  Checklist

 Enter a name for the checklist, up to 20 characters.
 Press F4 to see a list of existing checklists.

##  Enforce Order

 Select this checkbox to ensure that users can only complete checklist tasks and steps in order.

##  Enable Notification

 Select this checkbox to enable automatic notifications for the checklist. When you select this checkbox, the system sends messages to users assigned to tasks and steps with the Use Notification checkbox selected.

## Private

Select this checkbox so that only you can view, edit, or delete this checklist.
Note: Private checklists still display on the WF Checklists report.

##  Description

 Enter a description for the checklist, up to 2,000 characters.

##  Expected Completion

 Enter the expected completion date for this checklist. This field is for informational/administrative purposes and does not display anywhere else in the system.

## Date Completed

This field automatically defaults the date the last item task is complete. The system allows overrides to this field.
Note: The system does not update this field if you change the status of a checklist from final to another status and back again. In this case, you may want to override the default to ensure proper record keeping.

## Task

Enter N, New, or + to create a new task.
Press F4 to view existing tasks for this
 checklist.

##  Summary

 Enter a brief description of this task, up to 256 characters.

##  Required

 Select this checkbox if someone must finish this task to complete the checklist.

##  Use Notification

 Select this checkbox to have the system notify the assignee when the task is ready.

## Status

 Enter the initial status for this checklist,
 or press F4 to select from a list of available status codes.
If you see these codes: 999997, 999998, or 999999, you should not use them. Instead, set up status codes for New, In Progress, and Final in WF Checklist Status Codes.

##  Description

 Enter a description for the task, up to 2,000 characters.

## Task Type

Select the type for this task.

- Manual – This indicates the task occurs outside of the software.

- Form – This indicates the task involves a form in the software.

- Report – This indicates the task involves a report in the software.

##  Form/Report Name

 This field is accessible when you select either Form or Report as the task type. Enter the form or report that applies to the step or press F4 for a list.

## Assigned To

Enter the name of the user to assign this task to. Press F4 for a list of users.
If you do not specify a user, this task only displays on the WF Checklist Tasks form.

Related information

- [WF Checklist Tasks](/en/vista/vista/system-tools/work-flow/checklists/wf-checklist-tasks)

##  Assigned Date

 This field automatically defaults the current date when you specify a user in the Assigned To field. The system allows overrides to this field.

## Due Date

Enter the date the task must be complete by. This date displays with the task on WF User Tasks and WF Checklist Tasks.

Related information

- [WF Checklist Tasks](/en/vista/vista/system-tools/work-flow/checklists/wf-checklist-tasks)

## Completed By

When a user completes this task, this field automatically defaults the user name. The system allows overrides to this field.
Note: The system does not update this field if you change the status of a task from final to another status and back again. In this case, you may want to override the default to ensure proper record keeping.

## Completed Date

When a user completes this task, this field automatically defaults the current date. The system allows overrides to this field.
Note: The system does not update this field if you change the status of a task from final to another status and back again. In this case, you may want to override the default to ensure proper record keeping.

##  Step

 Enter ‘N’, ‘New’, or ‘+’ to create a new step for this task item.

## Summary

Enter a brief description of the step, up to 256 characters.

##  Required

 Select this checkbox if someone must finish this step to complete the task.

##  Use Notification

 Select this checkbox to have the system notify the assignee when the step is ready.

## Status

 Enter the initial status for this checklist,
 or press F4 to select from a list of available status codes.
If you see these codes: 999997, 999998, or 999999, you should not use them. Instead, set up status codes for New, In Progress, and Final in WF Checklist Status Codes.

##  Description

 Enter a description for the task, up to 2,000 characters.

## Step Type

Select the type for this step.

- Manual – This indicates the step occurs outside of the software.

- Form – This indicates the step involves a form in the software.

- Report – This indicates the step involves a report in the software.

##  Form/Report

 This field is accessible when you select either Form or Report as the step type. Enter the form or report that applies to the step or press F4 for a list.

## Assigned To

Enter the name of the user for step assignment. Press F4 for a list of users.
If you do not specify a user, this step only displays on the WF Checklist Tasks form.

Related information

- [WF Checklist Tasks](/en/vista/vista/system-tools/work-flow/checklists/wf-checklist-tasks)

##  Assigned Date

 This field automatically defaults the current date when you specify a user in the Assigned To field. The system allows overrides to this field.

## Due Date

Enter the date the step must be complete by. This date displays with the step on WF User Tasks and WF Checklist Tasks.
