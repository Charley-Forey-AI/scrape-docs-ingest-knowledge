# Field Definitions: VPMenu Form Properties Form

The following is a list of field descriptions for the VPMenu
 Form Properties form. Many of the descriptions include links to other topics that provide additional information about or related to the topic.

## All Form & Field Overrides

Check this box to reset all user form and
 field overrides (Field defaults & 'skip input' overrides, Form & grid display
 overrides, 'Required input' overrides, and Search on Enter or Tab). When
 checked, the individual options are automatically selected for reset.
Leave this box unchecked to manually select the user form and field overrides that you want reset.
Reset button
After checking some boxes in the Reset Options
 section, click the Reset button to apply the changes.
A message will display indicating that you are
 about to reset user level form and field overrides and warning you that the form will be
 reloaded and any pending record changes lost. Click Yes to reset, or No to
 cancel.
Once the reset is finished, the Form
 Properties form will close and the reset is effective immediately - you will not be
 required to close and reopen the form for the changes to take effect.

## Field Defaults & Skip Input Overrides

Check this box to reset the Default Type, Value, and Skip Input overrides (defined on the System Overrides tab of the [Field Properties](/en/vista/vista/system-tools/user-interface-guide/system-forms/about-the-field-properties-form) form) for all fields on the current form.
Leave this box unchecked to leave the system Default Type, Value, and Skip Input overrides intact during the reset.
Reset button
After checking some boxes in the Reset Options section, click the Reset button to apply the changes.
A message will display indicating that you are about to reset user level form and field overrides and warning you that the form will be reloaded and any pending record changes lost. Click Yes to reset, or No to cancel.
Once the reset is finished, the Form Properties form will close and the reset is effective immediately - you will not be required to close and reopen the form for the changes to take effect.

## Form, Grid Display, & Label Overrides

Check this box to reset the Show on Form, Show in Grid, Description in Grid, Form Label, and Include in Filter Form overrides (defined on the System Overrides tab of the [Field
 Properties](/en/vista/vista/system-tools/user-interface-guide/system-forms/about-the-field-properties-form) form) for all fields on the current form.
Leave this box unchecked to leave the system form, grid display, and label overrides intact during the reset.
Reset button
After checking some boxes in the Reset Options section, click the Reset button to apply the changes.
A message will display indicating that you are about to reset user level form and field overrides and warning you that the form will be reloaded and any pending record changes lost. Click Yes to reset, or No to cancel.
Once the reset is finished, the Form Properties form will close and the reset is effective immediately - you will not be required to close and reopen the form for the changes to take effect.

## Paging

On most forms you can enable Paging
 from the Form Properties
 menu.

### Enabling Paging

Check Enabled to enable paging on a form.
Enter or select the number of records to display per page on this
 form from the Page Size
 drop-down.Note: Set the Page Size for all forms in
 VA Site Settings >  Page Size.

### Page Size

With Paging enabled for a form, users can change the Page Size from the Paging drop-down on the form
 header.
Select the Paging drop-down from the form header and select the Pagination options for On or Off to enable or disable paging
 for a form on a user by user basis.

## Search on Enter or Tab

Restrict search results loading until a user enters Enter or Tab in the Search box.
Check this box when paging is enabled to restrict filtering the grid until
 the user enters Enter or
 Tab.
Leave this box unchecked to being filtering the grid as a search is being entered.
For more information on enabling paging, see [Paging](/en/vista/vista/system-tools/user-interface-guide/system-forms/about-the-form-properties-form/field-definitions-vpmenu-form-properties-form#concept-377--en).

## User's Default Tab

Using the drop-down menu, select the default
 tab for this form. This is the tab that will display each time this form is opened.

- 0=Grid Tab – Select this option to have the Grid tab always display
 when this form is opened. Useful if you prefer to always work in a grid format.

- 1=Info Tab – Select this option to have the Info tab always display
 when this form is opened. Useful if you prefer to always work in ‘entry form’ format
 (default format).

Note: Many forms have other tabs in addition to the Info and
 Grid tabs; however, you will only be able to set the default tab to either Grid or Info.
Note: You can reset this option manually or auto-reset it by
 checking the All Form & Field Overrides option when resetting user overrides. Reset
 button
Reset button
After checking some boxes in the Reset Options
 section, click the Reset button to apply the changes.
A message will display indicating that you are
 about to reset user level form and field overrides and warning you that the form will be
 reloaded and any pending record changes lost. Click Yes to reset, or No to
 cancel.
Once the reset is finished, the Form
 Properties form will close and the reset is effective immediately - you will not be
 required to close and reopen the form for the changes to take effect.

## User's Default Attachment Type

Enter the default attachment type for this
 user when attaching documents to records on this form. Press F4 for a list
 of valid attachment types.
If you specify an attachment type here, that
 type will default in the Attachment Type field on the Attachment
 Detail Form when the user is attaching a document.
See [Setting Default Attachment Types for Forms](/en/vista/vista/system-tools/document-management/dm-setup-and-maintenance/set-up-document-management/set-up-attachment-types/set-default-attachment-types-for-forms) for more information on setting up default attachment types.
