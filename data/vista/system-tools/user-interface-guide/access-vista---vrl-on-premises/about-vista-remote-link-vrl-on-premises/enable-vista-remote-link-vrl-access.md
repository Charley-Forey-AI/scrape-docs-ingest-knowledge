# Enable Vista Remote Link (VRL) Access

The steps you must complete so that your users can access the Vista client over the internet.
Review the VRL overview information found in [About Vista Remote Link (VRL) On-Premises](/en/vista/vista/system-tools/user-interface-guide/access-vista---vrl-on-premises/about-vista-remote-link-vrl-on-premises).

You can use Vista Remote Link (VRL) to enable the Vista™ client to access to the Vista™ server using an internet connection. Providing access over the internet requires use of a reverse proxy server in addition to the application and database server(s) so that only authorized users have access to the data. You must set up this system outside your domain's firewall but within the DMZ to allow the remote Vista™ client to connect securely.The main steps for enabling remote Vista™ client access for your users:

1. [Prepare for installation on the reverse proxy server.](/en/vista/vista/system-tools/user-interface-guide/access-vista---vrl-on-premises/about-vista-remote-link-vrl-on-premises/enable-vista-remote-link-vrl-access/prepare-for-installation-on-the-reverse-proxy-server)

1. [Download and run the executable on the reverse proxy server.](/en/vista/vista/system-tools/user-interface-guide/access-vista---vrl-on-premises/about-vista-remote-link-vrl-on-premises/enable-vista-remote-link-vrl-access/download-and-run-the-executable-on-the-reverse-proxy-server)
