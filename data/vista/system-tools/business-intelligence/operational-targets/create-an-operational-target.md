# Create an Operational Target

Follow the steps below to enter operational targets using the BI Operational Target form.
For example, you would like equipment usage to be 75%
 in 2019 and 80% in 2020, and you would like to measure equipment usage every month. You
 can also set up several different target levels that apply, for example a gold, silver,
 and bronze standard.Click [here](/en/vista/vista/system-tools/business-intelligence/operational-targets/operational-targets---overview) for an overview on using the operational
 target feature.

1. Open the [BI Operational Targets](/en/vista/vista/system-tools/business-intelligence/operational-targets/about-the-bi-operational-target-form) form in the Business Intelligence module.

1. Click the New Record icon () to create a new target. Targets can span multiple years. That means if the
 target that you are creating is equipment usage of 75% in 2019 and 80% in
 2020, you only need to create a single record.

1. Use the Target
 Name and Descriptionfields to enter
 general information about the target.

1. Optional: Define how the report will be grouped and filtered. To group or filter the report, select a view in the
 Target Type field and then use the Grouping
 Level, Grouping Value,
 Filter Field, and Filter Value fields to
 define how the report will be grouped and filtered.

1. Use the Target Type field to select the view that will be used to group and filter the report. The view selected in this field will apply to both the grouping and filtering. [More](/en/vista/vista/system-tools/business-intelligence/operational-targets/about-the-bi-operational-target-form/field-definitions-bi-operational-targets-form#ID-00009387--en)

1. Optional: Select
 the field that will be used to group the report in the Grouping Level field.

1. Optional: Select the field that will be used to filter the report in the Filter Field field, and then enter the value used to filter the report in the Filter Value field. When you create the custom report, you can actually use the values entered in the Grouping Level, Grouping Value, Filter Field, and Filter Value fields in any way. For example if you would like two grouping levels instead of a grouping level and filter option, you can set that up when you create the custom report.

1. Use the Actuals field to select the query that contains the actual amounts that you would like to compare to the target. [More](/en/vista/vista/system-tools/business-intelligence/operational-targets/about-the-bi-operational-target-form/field-definitions-bi-operational-targets-form#ID-000093d2--en)

1. Select the period of time that applies to the target, and how often you would like to measure it - for example 2014 through 2016, and reported on every month. Use the Begin Date and End Date fields to set the range of time that you would like to track progress, and then use the Period field to select how often you would like to measure progress.
The selection in the Begin
 Date, End Date, and Period field determines which fields will display on the
 Budget tab in the lower portion of the form. That is where the target values
 are entered.

1. Use the Revision field in the lower portion of the form to create a new
 revision. [More](/en/vista/vista/system-tools/business-intelligence/operational-targets/about-the-bi-operational-target-form/field-definitions-bi-operational-targets-form#ID-0000941f--en)

1. Optional: Use the Target Level field on the Info tab in the lower portion of the form to associate the target with a target level set up using the BI Operational Target Level form. [More](/en/vista/vista/system-tools/business-intelligence/operational-targets/about-the-bi-operational-target-level-form)

1. Use the Budget tab in
 the lower portion of the form to enter the target amounts. The years and periods
 that display on this tab are calculated using the Begin
 Date, End Date, and Period fields on the Info tab in the upper portion of the form.


1. Click the Save icon () at the top of the form to save the new operational target.

1. Once the target information has been entered, make sure a query has been selected in the Actuals field and then create a custom report to display both the targets and the actual values. [More](/en/vista/vista/system-tools/business-intelligence/operational-targets/operational-targets---overview)
