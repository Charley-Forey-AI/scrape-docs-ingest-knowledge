# My Subcontract Billing Entry Field Descriptions

Use the table below for reference when completing the fields in the My Subcontract Billing Entry screen. You can enter values in the dark gray columns on the screen as well as attach files with the Slider Bar. The remaining fields are all informational.
Fields/Buttons
Descriptions

Submit Billing button
After completing the screen and reviewing the progress billing, click this button to generate the unapproved invoice and submit it for review and eventual payment.

- Enter the Invoice # or leave blank to let Spectrum auto-assign the next number.

- If an 'Action Type #3' Document Tracking item is set up for the Subcontract or Vendor, the new invoice will be placed on hold.

- When prompted, if you select Continue to proceed with the billing,
 the [Unapproved Billing Report Sample Report](/en/spectrum/spectrum/accounting/subcontract-kiosk/my-subcontract-billing-entry/unapproved-billing-report-sample-report) will be generated.
Note: Consider adding your Subcontract Administrator to the first reviewer
 position on the list. By routing the invoice to the
 Administrator, they can review that the invoice has been
 completed properly and all supported documentation is
 included before it goes down your standard approval
 paths.

Exit/Delete buttons

- Click Exit to save your changes and leave this screen without running the
 update and creating an invoice. More info: An Exit button
 has been provided for clarity as the typical Subcontract
 Kiosk user won't be a Spectrum user. Also, we have added
 additional protection to prevent the user from accidentally
 not saving their work. All data entry is automatically saved
 whenever the user exits the screen using the Exit button,
 clicks the 'x' on the Spectrum tab or clicks the checkmark
 icon.

- Select a grid line and click Delete to remove a billing item. If a billing is deleted, the same billing number will re-default if you proceed to start the billing again (assuming it hasn't been used in the meantime in Invoice Entry).

Billing number
No changes are allowed in these fields.
The billing number value came from the 'Next payment number' when the current billing was initiated.

Total amount (this billing)

Total before tax
The amount total will reflect whether a subcontract is fully-billed, including any unposted invoices or credit memos assigned to the subcontract. The system will check the dollar amount entered and display an error message if the amount entered exceeds the revised contract amount in the subcontract file. If the Accounts Payable Installation checkbox for subcontract stop payment is selected, then amounts in excess of the revised contract amount will be disallowed.
When two-step purchase order receiving is used, the total before tax amount will default to the packing list total.

GST

VAT code
These fields display only when the 'Utilize value added tax (VAT) tracking?' checkbox in General Ledger Installation is selected.

Total amount
The combined total, including sales tax, displays. No changes are allowed.

Subcontract progress

Total contract amount
This is the sum of 'Contract amt' column in grid; this is also the same amount appearing on the 'My Subcontract Manager' screen.

Amount to-date

Remaining to bill
This is the 'Total contract amount' minus 'Amount to-date'.

Retention (this billing)

Subject to retention

Holdback percent
This rate is not locked in until you click the Submit Billing button.

Retention amount
This is the 'Subject to retention' amount multiplied by the 'Retention percent' / 100.

Grid fields that allow changes

Qty this period
 When you enter a value here, the system recalculates the 'Amt this period', 'Qty to-date', 'Amount to-date' and 'Tax amount' accordingly.

Amt this period
When you enter a value here, the system recalculates the 'Qty this period', 'Qty to-date', 'Amount to-date' and 'Tax amount' values.
Protection is built in to prevent entry of values greater than what is currently on your subcontract. The system reviews all posted, unposted and unapproved billings to verify that the subcontract total has not been exceeded.
Note: Vendor Invoice Entry will correspondingly be modified to additionally
 read for billed amounts on the subcontract.
