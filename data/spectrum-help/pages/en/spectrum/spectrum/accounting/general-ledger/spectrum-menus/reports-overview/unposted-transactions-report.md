# Unposted Transactions Report

View how many unposted transactions exist in each Spectrum module.
This report is useful for identifying any outstanding transactions that must be updated before they can be reconciled. For example, controllers may use this audit report at month-end to ensure that all work-in-progress has been posted before proceeding with G/L month-end close and financial reporting. The report includes contract records using scheduled dates through the current period that have not been updated (earned) using Earned Revenue Processing. Contracts with an Earned revenue basis of 'As billed' are not included on the report.
