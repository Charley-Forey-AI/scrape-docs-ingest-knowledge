# G/L Processing Date Change - Field Descriptions

Use the table below for reference when completing the fields on this screen.
Fields
Descriptions

Current settings

Processing date
Enter the new processing date or click the date on the calendar. This will be the default for entries, inquiries, and reports. The new date must be between the minimum and maximum date range. Click Reset to restore the current setting.
Note: Each module has its own Processing
 Date screen where you can modify the current date. This date
 must fall within the module's minimum and maximum processing dates.

Fiscal year
The fiscal year that the processing date falls within displays.

Period
The period number that the processing date falls within (number 1-13) displays.

Period start date
The first day of the period for the specified processing date displays.

Period end date
The last day of the period for the specified processing date displays.

Allowable entry range

Minimum date
The earliest date that a processing date can be changed to. This date is set
 in the System Administration > Installation > Processing Date Change screen.

Maximum date
The latest date that a processing date can be changed to. This date is set in
 the System Administration > Installation > Processing Date Change screen.

Reset
Click to reset the current processing date to the date that was set up in the Installation > Processing Date Change screen.
