# My bill-to-date incorrectly shows $0.00 in my draw request. Why?

After adding the first draw request for a contract, subsequent draw requests should be created by updating the prior draw request. This causes the next draw request number to display automatically in the draw request field. If you mistakenly add a draw request other than the first one for the contract, it will not reflect the previously billed amounts. To correct this, the legitimate draw request can be reversed or re-updated.
