# Prevailing Wage - Field Descriptions

Use the table below for reference when completing the fields on this screen.
Fields/Buttons
Descriptions

Health & welfare
Click the drop-down arrow to select a valid health & welfare code. After making your selection, the corresponding code description and type display. Up to 10 characters are allowed.

Pension
Only add-ons not paid to the employee on the check are permitted. Remember that pay group add-ons are not taken on private jobs.
The ability to charge prevailing wage adjustment benefits to a non-cash add-on code allows employers to pay excess fringe amounts directly to a third party.
Entry of an add-on code with a Workday user-defined formula or calculation method of non-stated fringe will be disallowed.
Refer to the [Example: Fringes paid to a third party](/en/spectrum/spectrum/accounting/payroll/maintenance-overview/non-union-pay-group-maintenance/prevailing-wage/example-fringes-paid-to-a-third-party) to see calculation examples.

Cash fringe benefit
 Only add-ons not paid to the employee on the check are permitted. A Cash fringe benefit code is not entered if the Pension code is a prevailing wage adjustment. Up to 10 characters are allowed.
Entry of an add-on code with a Workday user-defined formula will be disallowed.

Variables buttons
These buttons are only enabled if the pension and/or cash fringe use a user-defined formula. Click these buttons to access the values for this formula.

Fringe rates

Level
Base
The standard pay rate for this pay group displays. To change these rates, close the Prevailing Wage window and then edit the rates for levels 1-9 in the Non-Union Pay Group Maintenance screen.

Fringe
Full
Enter the fringe amount here. Fringe amounts are payments made in excess of the benefits paid on the employee's behalf.
In the Full column, the sum of the Base pay rate and the Fringe pay rate display. No entry is allowed.

Disable DB/PW benefits?
This checkbox is only available if the prevailing wage codes are completed. Select this checkbox to post the time card line as a private job and to ensure the employee will be paid based on the base pay of the pay group (no fringe, prevailing wage, prevailing wage adjustment or pension would apply). Any company 401(k) employee and employer amounts will be calculated, as they would be for a private job.

Disable certified?
Select this checkbox to designate the time card line as non-certified; for example, you might set this flag at certain levels for jobs that do not have any fringe amounts set up. This checkbox can be selected even if the job and phase indicate the line is certified, or if Davis-Bacon and prevailing wages are not applicable. [Time Card Entry Certified Hierarchy](/en/spectrum/spectrum/accounting/payroll/default-hierarchies-overview/time-card-entry-certified-hierarchy)
