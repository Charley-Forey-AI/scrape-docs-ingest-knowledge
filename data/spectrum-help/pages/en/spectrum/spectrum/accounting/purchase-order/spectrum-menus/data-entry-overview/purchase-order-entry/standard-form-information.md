# Standard Form Information

Use this window to enter payment and shipping terms, as
 well as order information for this purchase order.
Once you have completed entering data in this window, click OK to return to the Purchase
 Order Entry screen.
Field/Button
Description

Remarks
Enter any remarks for this Purchase
 Order, or enter a valid remarks code already set up in P.O. Remarks
 Maintenance.
Special Instructions
Enter any special instructions for the
 Purchase Order, or enter a valid special instruction code already set up in the
 Special Instructions Maintenance.
Form status
The purchase order print status will
 display in this field.
Shipping details

Ship-to address
Select an entry method for the ship-to
 address, and then enter the corresponding address below.

- All work order types will have 'Default ship-to
 address', 'Warehouse address', and 'Input address' entry method options.


- If the 'PO Type' is Job, the job address will
 default in this field.

- If the 'PO type' is Warehouse, the warehouse address
 will default in this field.

- For Work order purchase orders, an additional 'Work
 order site address' entry method option is available.

- For Job and Work
 order purchase orders, when 'Warehouse address' is selected the Search
 Warehouses window will open to select the address.

Ship via
The ship via information displays in
 this field. This code must have been previously defined in the Ship Via File
 Maintenance screen.
Shipping terms
Press Enter to accept the default
 shipping terms as defined on the Purchase Order Installation screen, or enter
 the shipping terms for this Purchase Order
FOB location
Press Enter to accept the default
 from Purchase Order Installation, or enter the appropriate f.o.b. location for
 this Purchase Order. Examples of free-on-board locations are job site, truck,
 dock, factory, rail, barge, rafted, and so forth.
Purchase location
To use an alternate vendor address,
 enter a valid location code here or select one from the available search
 window. (This field only displays if the vendor has alternate locations set
 up.)
Tracking

Ordered by
Enter the name of the person who placed
 the order.
Confirmed by
Enter the name of the person who
 confirmed the order, or press Enter to accept the name which defaults, as defined on the
 Purchase Order Installation screen.
Order date
Enter the order date for tracking
 purposes. The date will default from the current purchase order processing date
 when adding a new purchase order.
Batch code
Enter the batch code for tracking
 purposes. The default operator code will default when adding a new purchase
 order.
Payment terms

Payment due basis
Select the payment due terms for this
 purchase order: Invoice
 date or First of
 next month.
Number of days
If the payment due terms are figured
 from the invoice date, enter the number of days allowed (for example, "30" for net
 thirty days terms). If the payment due terms are figured from the first of the
 month, enter the date that payments are due (for example, "10" indicates that payment
 is due on the tenth).
Discount due basis
Select the discount terms for this
 purchase order: Invoice
 date or First of
 next month.
Number of days
If the discount terms are figured from
 the invoice date, enter the number of days allowed. If the discount terms are
 figured from the first of the month, enter the number of days allowed for a
 discount.
Discount %
Enter the discount actual percentage.
 It is not necessary to convert the percentage to a decimal before entry. For
 example, if a five percent discount is allowed, type 5 and then press Enter; if
 a ten and sixty-four one hundredths percent discount is allowed, type 0.64 and
 press Enter.
Terms description
Press Enter to accept the terms
 description; otherwise, enter the payment terms for this Purchase Order.
Specify 'For Resale' on P.O. form?
When adding a new purchase order, this
 checkbox will be selected if the vendor's sales/use tax code is a 'Use' type
 code. If the vendor's sales/use tax code is a 'Sales' type code, this checkbox
 will default to blank.
Payment location
This field only displays if alternate
 vendor locations are set up. When adding a purchase order, the code designated
 as the default payment location displays.
