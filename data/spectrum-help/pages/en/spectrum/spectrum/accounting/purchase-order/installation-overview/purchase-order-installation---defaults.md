# Purchase Order Installation - Defaults

Use this tab to select default settings for the Purchase Order module. These settings can be changed as needed at any time.
Field/Button
Description

Default ship via
The system is prompting for the default
 response to the Ship
 via field during Purchase Order Entry. Examples of possible Ship
 via responses include UPS-Red, UPS-Blue, UPS-Surface, Will Call, Parcel Post,
 and Company Truck. Entry in this field will serve as a
 default for Purchase Order Entry, and may be overwritten or removed during
 entry of any given purchase order. It is recommended that you enter the most
 common 'ship via' response used on purchase orders by this company in this
 field. This field may be left blank if no default is desired.

Default ship terms
The system is prompting for the default
 response to the Ship
 terms field during Purchase Order Entry. Examples of possible
 Ship terms include Prepaid and Collect; shipping terms represent how the
 shipper will be paid. Entry in this field will serve as a
 default for Purchase Order Entry and may be overwritten or removed during
 entry of any given purchase order. It is recommended that you enter the most
 common Ship terms response used on purchase orders by this company in this
 field. This field may be left blank if no default is desired.

Default f.o.b.
The system is prompting for the default
 response to the F.O.B
 location field during Purchase Order Entry. Examples of possible
 F.O.B location responses include Job Site, Our Warehouse, Your Warehouse, and
 the name of a city. Entry in this field will serve as a
 default for Purchase Order Entry and may be overwritten or removed during
 entry of any given purchase order. It is recommended that you enter the most
 common F.O.B. location response used on purchase orders by this company in
 this field. This field may be left blank if no default is desired.


Default tax code
The system is prompting for the default
 response to the Sales/use
 tax field during Purchase Order Entry for purchase orders not
 for resale. Entry in this field will serve as a default
 for Proposed Purchase Order Entry and Purchase Order Entry if a use/sales
 tax code is not otherwise specified in the vendor or job file (based on Job
 Classification setup).
This field accommodates tax
 codes that are up to 15 alpha-numeric characters in length.

Default confirmed by
The system is prompting for the default
 response to the Confirmed
 by field during Purchase Order Entry. Examples of possible
 Confirmed by responses include the name or title of a key individual within
 your company. Entry in this field will serve as a default
 for Purchase Order Entry, and may be overwritten or removed during entry of
 any given purchase order. It is recommended that you enter the most common
 Confirmed by response used on purchase orders by this company in this field.
 This field may be left blank if no default is desired.

Default shipping address
The system is prompting for the default
 response to the Ship-to address fields during Purchase Order Entry for orders
 not designated for job-site delivery. This is the most common address you want
 the shipper to use when preparing deliveries other than to job sites. Entry in these fields will serve as the default for
 Purchase Order Entry, and may be overwritten or removed during entry of any
 given purchase order. It is recommended that you enter the most common
 Ship-to address
 used on purchase orders by this company in this field. These fields may be
 left blank if no default is desired.

Override default to ship-to-job or job address for job PO type?
Select this checkbox to always default
 the job address as the Ship-to address on Job PO's.
Override default to warehouse address for warehouse PO type?
Select this checkbox to always default
 the warehouse address as the ship-to address on Warehouse PO's.
Default type
The system is prompting you to select
 the Lump sum or
 Unit price option,
 depending upon whether by default your purchase orders are more frequently unit
 priced or lump sum entry. Any given purchase order may be either lump sum or
 unit price, as needed. Entry in this field will default during Purchase Order
 Entry, and may be over-written if desired.

- Unit price purchase orders require entry of quantity
 and unit price on each line of the PO, and the extension is then computed
 by Spectrum.

- Lump sum purchase orders may be issued; under this
 method, the operator simply enters the lump sum amount without regard to
 quantity and price per unit.
