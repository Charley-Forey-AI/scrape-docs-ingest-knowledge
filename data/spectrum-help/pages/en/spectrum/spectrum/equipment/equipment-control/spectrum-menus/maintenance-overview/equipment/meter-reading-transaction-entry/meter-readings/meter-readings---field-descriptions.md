# Meter Readings - Field Descriptions

Use this table for reference when completing the fields on this screen.
Fields
Descriptions

Batch
The operator's logon id defaults. Press Enter to accept the default, or type a batch code to identify the readings recorded at this time.

Equipment

Date
The date defaults from previous entry when the New window is used to enter multiple readings successively; otherwise, the date defaults to the current Equipment Control processing date.

Remarks
Enter any pertinent notes regarding this entry. Up to 50 characters are allowed. While working in one session, the remarks line will default from the previous entry.

Meters

Meter type
The meter type specified in the Equipment screen displays. If any of the four meter readings do not currently have a meter type specified in Equipment, then this column displays None, and the corresponding columns are blank.

Meter description
The meter description defaults from the Equipment screen.

Current reading
Enter the current meter readings for the specified equipment. Up to four meter readings can be recorded.
If the meter type for this piece of equipment is Miles or Kilometers, enter the odometer reading for this piece of equipment. If the meter type for this piece of equipment is Hours or Usage, enter the number of hours for this piece of equipment. The units (miles, kilometers, or hours) display from Equipment.
This field is disabled when no meter type is specified.
Note: Press Enter
 after completing this field to save the current entry and return the cursor
 to the beginning so you can begin your next meter reading.

Prior meter reading
This column displays the most recent meter reading prior to the date in the window heading. The order in which this entry is typed into the system is not relevant to the calculation. Instead, the comparison is to the stored dates. In most cases, the prior and last readings are likely to be the same date, but not in all cases.
Example:
Say that meter readings are entered daily and the operator has recorded meter #1 up to 12/04/xx. He then discovers that the reading dated 11/15 was accidentally not entered. The operator adds a reading entry dated 11/15/xx; the 'prior meter reading' will show the 11/14 reading and the 'last meter reading' column will show the one for 12/04. The warnings regarding 'normal usage' are always computed based on the prior reading.

Last meter reading
This column displays the latest meter reading on file in the system. The order in which this entry is typed into the system is not relevant to the calculation. Instead, the comparison is to the stored date. In most cases, the prior and last readings are likely to be the same date, but not in all cases.
Example:
Say that meter readings are entered daily and the operator has recorded meter #1 up to 12/04/xx. He then discovers that the reading dated 11/15 was accidentally not entered. The operator adds a reading entry dated 11/15/xx: the 'prior meter reading' will show the 11/14 reading and the 'last meter reading' column will show the one for 12/04. The warnings regarding 'normal usage' are always computed based on the prior reading.
