# Why Writing to Spectrum Tables is NOT Recommended

Spectrum programs contain extensive protection against
 out-of-balance conditions (for example, debits must equal credits) and other types of errors.
 This protection is programmed into the data entry screens and prevents users from accidentally
 entering in information that does not make sense (for example, trying to enter a job phase
 that has not been set up).
If Spectrum tables are written to with Info-Link, all of this protection
 is bypassed and any information can be entered in any column.
