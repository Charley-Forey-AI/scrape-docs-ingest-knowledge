# Time Card Approval - Reviewing the list

You can configure the Time Card approval workflow through
 the use of lists.
Each company can be configured in the [Time Card Approval Workflow Definition](/en/spectrum/spectrum/tools/workflow/workflow-definition-field-descriptions/time-card-approval-workflow-definition) screen to require Employee review OR Job review OR both approvals, and these selections will determine which records display on this screen:

- The Employee's Supervisor will see a list of employees assigned to review. The list can be sorted by Employee, Job, Phase or Date. The Job column will conditionally display 'Pending approval' if one or more employee time card records in the batch have yet to be approved by the Project Manager. These Reviewers will likely review the employee's time cards as each pay period ends, although it will be possible to approve Workflows in shorter time frames. Occasionally, an employee may have no pre-time card entered for the pay period. If the Payroll Department wants verification that it is correct that the employee did not work, a pre-time card can be entered with pay type 'U' (unpaid) and then routed for approval.

- The Job contact, for example, 'Project Manager', will see a list of jobs and phases assigned to review. These Reviewers will likely review the job time cards on a daily basis, although it will be possible to approve Workflows in longer time frames.

The system will further determine which records to display based on the item type (for example, only 'Rejected' records will display if accessing this screen from the [Time Card Approval Workflow Overview](/en/spectrum/spectrum/tools/workflow/workflow-definition-field-descriptions/time-card-approval-workflow-definition/time-card-approval/time-card-approval-workflow-overview) window and clicking the View All Rejected button.)
