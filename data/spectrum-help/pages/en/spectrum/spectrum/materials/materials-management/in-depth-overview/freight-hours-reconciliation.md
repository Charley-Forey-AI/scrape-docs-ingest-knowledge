# Freight Hours Reconciliation

The Freight Cost Reconciliation screen serves two purposes. The first is to adjust or reconcile the freight charged on a ticket-by-ticket basis. The second is to group together tickets and to create an A/P invoice. This is used after the vendor has sent out an invoice. This screen is driven by the vendor code, since it will be using the vendor code when the invoice is created.
However, when reconciling the hours for tickets, the "grouping" of tickets for invoicing is not of concern. The Freight Hours Reconciliation entry screen is available specifically for reconciling the quantity (hours or tonnage), for the tickets. [Freight Hours Reconciliation](/en/spectrum/spectrum/materials/materials-management/spectrum-menus/data-entry-overview/freight-cost-reconciliation/freight-hours-reconciliation)
The batch code can be used as a way to group tickets that have been reconciled based on hours. The batch code is a selection criteria in the Freight Cost Reconciliation entry screen. For example, you might designate that the batch code is assigned the date. Then in the Freight Cost Reconciliation entry screen, all tickets for that batch could be selected for payment by simply selecting that batch. Since the batch selection will be a SuperSelect, a range of batches (in effect, a range of dates) could be selected for payment.
Both hours and per-ton freight charges can be adjusted in this screen. Click the Preview button to preview a report for this screen. On this report, you can choose to list all tickets that have not yet been processed in the quantity reconciliation or updated to A/P in the freight reconciliation process (select Open). This provides a list of outstanding tickets that need to be processed.
