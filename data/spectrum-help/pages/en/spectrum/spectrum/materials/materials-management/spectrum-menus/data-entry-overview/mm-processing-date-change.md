# M/M Processing Date Change

This screen is used to change the date under which this
 module operates. The processing date must be set separately within each application being run
 on the software.
Field
Description

Processing date
The left column displays the current processing information. The information is re-displayed in the right column. Enter the new processing date by typing over the displayed date. It is not necessary to separate the month, day, and year with slashes; the system will insert them automatically.
If the new date is before the Minimum Processing Date or after the Maximum Processing Date established during the installation procedures, the following error message displays:
DATE ENTERED IS NOT WITHIN THE CURRENT MINIMUM AND MAXIMUM DATE RANGE
To leave the processing date as it is displayed, enter / and press
 Enter to exit. To change the date, enter a date
 within the specified date range.

Fiscal year
The fiscal year in which the requested date is located displays.

Period
The General Ledger period in which the requested date is located (number 1-13) displays.

Period date
The start and end dates of the period in which the requested date is located displays.

Minimum date
The earliest date that a processing date can be changed to is displayed in
 this field. This date is set in the Admin > Installation > Processing Date Change screen.

Maximum date
The latest date that a processing date can be changed to is displayed in this
 field. This date is set in the Admin > Installation > Processing Date Change screen.
