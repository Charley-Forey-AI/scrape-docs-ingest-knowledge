# Projected Cost Entry by Group - Field Descriptions

A reference for completing and interpreting the fields in the Projected Cost Entry screen.
FieldsDescriptions
JobEnter a valid job code, and the job description will display to the right. During the Projected Cost Update, if a job is selected for an updated batch while it is also in use by another operator in Projected Cost Entry, the specified job will not be updated at that time. Instead, the job will remain built and will be available for update after the current entry is complete.
BatchThe batch code associated with the specified job displays. This is the batch code assigned when the job was built. Select the magnifying glass icon adjacent to this field to view the [Projected Cost Batch Inquiry](/en/spectrum/spectrum/project-management/job-cost/spectrum-menus/data-entry-overview/projected-cost-overview/projected-cost-entry-by-group/projected-cost-entry-by-group---field-descriptions/projected-cost-batch-inquiry)
Projected profit at completionThe job's projected profit amount displays. Projected profit is defined as Revised contract (or Projected contract for unit price jobs) less Projected costs.
Projected cost at completionThe current projected cost total for the selected transactions display.
Variance since last projectionThis is the variance for the selected phase line based on the variance of the current projection (at completion) and the current estimate and last projection. [Variance Calculations for Projected Costs](/en/spectrum/spectrum/project-management/job-cost/spectrum-menus/data-entry-overview/projected-cost-overview/projected-cost-entry-by-group/projected-cost-entry-by-group---field-descriptions/variance-calculations-for-projected-costs)
Buttons
More TotalsSelect to view the [Summary Totals](/en/spectrum/spectrum/project-management/job-cost/spectrum-menus/data-entry-overview/projected-cost-overview/projected-cost-entry-by-group/projected-cost-entry-by-group---field-descriptions/summary-totals) window.
ExpandSelect to display the following additional entry fields for each line item: Projected hours, Percent of hours to complete, Hours to complete, and Estimated hours.
DetailSelect to view the group phase detail information for the selected transaction line.
Cost-to-dateThis window displays the job-to-date cost, open commitments, and totals.
EstimateThe [Phase Estimates window](/en/spectrum/spectrum/project-management/job-cost/spectrum-menus/data-entry-overview/projected-cost-overview/phase-estimates-window) displays the original estimate, changes and revises estimates figures for this group.
Grid
LineSequential line numbers are assigned and maintained by the software.
GroupThis group defaults from the Group summary selection in the Build Projected Cost Transaction screen. If Major group only or Through minor group is chosen, then this screen will show one line for each group. When a percent complete, cost-to-complete or final projection is entered for the group, the software will allocate this percent to all detail phases in the group.
DescriptionThe major group, minor group, or phase group descriptions display based on your selection in the Build Projected Cost Transaction File screen. If a phase category has been designated in the Jobs screen, then the software will use the category-specific group description. If no phase category has been designated, the company-wide description will display.
JTD + openJTD+ open - This field displays if the Include open commitments checkbox in Build Project Cost was selected when the job was built. The open commitment calculation will include use tax recorded on purchase orders and job requisitions.
JTD - This field displays if the Include open commitments checkbox in Build Project Cost was not selected when the job was built. The open commitment calculation will include use tax recorded on purchase orders and job requisitions.

EstimateThe current estimates for this phase or group display.
% completeEnter the new percent complete, as recorded on the project manager's Projected Cost Worksheet. The software will automatically apply this percentage with the job-to-date actual amount to compute the projected total cost. If the percent complete is 100%, then Cost-to-complete is automatically zero.
Cost/hours to completeEnter the new cost or hours to complete, as recorded on the project manager's Projected Cost Worksheet. The software will automatically add this amount to the job-to-date actual cost to compute the projected total cost. This is automatically zero if the percent complete is at 100%.
Projected cost/hourEnter the new projected cost or hours, as recorded on the project manager's Projected Cost Worksheet. Entry in this field will become the new projected cost for this phase/cost type.
HoursOn each line, the hours for the JTD + open, Estimate, Percent complete, Cost/hours to complete, and Projected cost/hour fields display. This field displays in Expand mode.
Job totalsThe column totals from the line transactions display.
