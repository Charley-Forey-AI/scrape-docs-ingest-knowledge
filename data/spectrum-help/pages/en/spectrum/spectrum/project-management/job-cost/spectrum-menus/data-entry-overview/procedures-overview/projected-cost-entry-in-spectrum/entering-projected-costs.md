# Entering Projected Costs

Once the Projected Cost file has been built, the jobs included in the build are now accessible in the Projected Cost Entry screen.
Projected cost entries can be made by changing the Percent Complete, Cost/Hours to Complete, or Projected Cost/Hours at Completion. These areas are all related, and entries into one of the fields automatically updated the other two fields.
The initial calculations of the fields are:

- Percent Complete – job-to-date costs / projected cost at completion

- Projected Cost – current projected cost from the phase file

- Cost to Complete – projected cost less job-to-date costs

There is an option on the Job Cost Installation screen to set projected costs equal to actual costs during the build, if the actual costs exceed the current projected costs. If this checkbox is selected, phases meeting this criterion are automatically set to 100% complete during the build.
