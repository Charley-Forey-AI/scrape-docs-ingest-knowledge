# Labor Analysis Report - Field Descriptions

Use the table below for reference when completing fields on this screen.
Fields
Descriptions

Selections

Job
Enter the number of the job to be included on this report.

Phase
Enter the code of the phase to be included on this report.

Labor cost type
Enter the cost type codes corresponding to the labor Cost type to be included
 on this report. This defaults to the labor cost type designated in the
 Job Cost Installation screen. Up to five codes may
 be entered.

Division
Enter the Division to which jobs on this report should belong.

Superintendent Estimator Project manager
Enter the superintendent, estimator, and project manager of jobs to be included on this report.

Through period end date
Enter the period end date, or press Enter for current Job Cost processing date.

Fiscal year
This is a display-only field that appears based on the period-end date.

Period
This is a display-only field that appears based on the period-end date entered above.

Use projected cost?
Select to include Projected cost on the report. The default is controlled by
 the Show projected
 costscheckbox on the Job Cost
 Installation screen.

Include group subtotals?
Select to include group subtotals, or leave clear to print the report without
 group subtotals. The default is controlled by the Show group subtotals field
 on the Job Cost Installation screen.

Include Payroll in progress?
Select to include pre-time card entries and time card entries in the current pay cycle on this report.
If you use this feature and post Payroll burdens to separate phases, verify that the burden phases are added to the phase detail file before any pre-time cards are entered using these phases.
When this checkbox is selected, Spectrum computes the total payroll cost
 (including burdens) that has been entered into Payroll > Pre-Time Card Entry but has not yet been calculated or posted. This cost is added
 to the actual cost of each job.
Note: This is only an estimate and does not reflect the actual
 computed costs. If the job uses actual burdens, the burden cost is estimated
 using the [Job Payroll Setup](/en/spectrum/spectrum/project-management/job-cost/spectrum-menus/jobs/job-properties/job-payroll-setup) > Payroll burden section. In addition, if the payroll information is changed
 before the payroll is calculated and updated, the actual costs that post to
 the job will be different from what is reflected on the previously printed
 report. Estimated payroll costs are also included in the current week/month
 on this report.

Include A/P contract labor hours with Payroll hours?
Select this checkbox to include Accounts Payable contract labor hours with Payroll hours on this report.

Report format

Standard
Expanded with week-to-date detail
Expanded with month-to-date detail
Expanded with year-to-date detail
Click Standard or one of the Expanded options (week-to-date, month-to-date, year-to-date). When you click Expanded, quantity figures are included on the report.

Week-to-date begin
If the Week-to-date option is selected in the Period type radio group, enter the first date to include on the report or press Enter to accept the system default, which is 6 days less than the date that displays in the Week-to-date end field.
Click the Date Calculator icon to add a date calculator
 to the report filter parameters.

Week-to-date end
The current Job Cost processing date displays. If the Week-to-date option is
 selected in the Period
 type radio group, press Enter to accept the system
 default.
Click the Date Calculator icon to add a date calculator
 to the report filter parameters.

Job status

Active?
Inactive?
Complete?
Select at least one Job status to include on the report.
