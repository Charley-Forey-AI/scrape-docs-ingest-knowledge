# Reclassify Job Cost History: Transaction Type AP - Vendor

The G/L codes used in these transactions will default just as if the Job Cost transactions were entered manually. If these codes do not default, they can be entered manually. An error report will show any records without G/L codes.
Overhead (OH) and General Ledger (GL) transactions in Job Cost History will be converted to Job Cost transactions when J/C History Reclassification is used. These types of transactions originate in other modules (PR and GL respectively) and therefore are not otherwise accessible for Job Cost Transaction Entry. By converting theses records to Job Cost types, it will now be possible to transfer all costs for a job using this screen.
When job-based security is in use, this feature will only display jobs for which the operator has access, but it will still be possible to transfer costs to jobs outside the accessible range. The job-based security is in place to prevent viewing of sensitive information. If the operator should not be allowed to reclassify costs to certain jobs, access to this function should be removed.
