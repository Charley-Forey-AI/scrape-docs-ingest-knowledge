# Reclassify Job Cost History Update

This screen is used to confirm and update the changes made to Job Cost Transaction Entry. The changes can be updated, more edits can be made, or the transaction entry update can be canceled.
Protection is incorporated into the update to prevent posting transactions where G/L code and job division do not match (based on the Job Cost Installation). In this case, a warning message displays alerting the operator that an error report has printed and the transaction will not be updated. The update also verifies that G/L code and cost type combinations are correct. If the G/L code and the cost type combinations are not correct, a warning message will display on the screen alerting the operator that an error report has printed and the transaction will not be updated.
After clicking Update, an authorization to update message displays. Click Continue to perform the update. If you have security authorization, the confirmation will include a link to the [Cost Transaction Report](/en/spectrum/spectrum/project-management/job-cost/spectrum-menus/data-entry-overview/job-cost-transaction-entry/cost-transaction-report)
