# Job Cost Transaction Entry - Cost Center Information

If the cost center feature is enabled in the Enterprise Installation screen, when entering a job Spectrum allows transaction entry for the job only if you have permission to access the job. As you request each job, Spectrum compares the cost center assigned to the job with cost centers in your assigned scheme; if the cost center is not included, then entries for that job will not be allowed. In addition, Spectrum compares the cost center assigned to each phase, if applicable, with cost centers in your assigned scheme; if the cost center is not included, then that phase cannot be recorded.
When your scheme includes override settings for job entries in Cost Center Scheme Maintenance, the Job Cost Transaction Entry screen will validate the cost center assigned to transaction detail lines based on these overrides. The override cost center(s) supersede the cost center list defined for the scheme in general. Spectrum compares the cost center assigned in the entry screen detail with job override cost centers in your assigned scheme; if the cost center is not included, then the transaction entry line will not be allowed.
As each transaction line is added, at the Debit G/L account field, Spectrum displays the cost center assigned to the job (or phase, if applicable). The cost center is assigned automatically and cannot be changed by the operator. When adding records Spectrum verifies that the debit cost center assigned to the transaction entry line is permitted for that G/L account code by comparing the entry to the list of allowed cost centers for that G/L account. Validation is also performed if you attempt to change the debit G/L account or delete an existing transaction entry line. Spectrum also compares the debit G/L account's list of shared cost centers with cost centers in your assigned scheme; if no cost centers are common to both, then that G/L account code is not allowed.
As each transaction line is added, at the Credit G/L account field, Spectrum defaults the cost center assigned to the job (or phase, if applicable) and permits you to override the entry. In Add mode, Spectrum verifies that the credit cost center assigned to the transaction entry line is permitted for that G/L account code by comparing the entry to the list of allowed cost centers for that G/L account. Validation is also performed if you attempt to change the credit G/L account or cost center, or delete an existing transaction entry line. Spectrum also compares the credit G/L account's list of shared cost centers with cost centers in your assigned scheme; if there are no common cost centers, then that G/L account code is not allowed.
For non-stock items, Spectrum will verify that the cost center exists and the Operator is authorized to that cost center and the credit G/L account code. No cost center overrides are available for non-stock items.
