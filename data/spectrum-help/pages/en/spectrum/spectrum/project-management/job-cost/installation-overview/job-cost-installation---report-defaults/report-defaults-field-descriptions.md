# Report Defaults Field Descriptions

Use the Report Defaults field descriptions when completing
 actions on this screen.
Fields
Descriptions

Include open commitments?
Entry in this field will determine whether or not
 the open commitments will be included in the job-to-date cost amount
 on Job Cost reports and inquiries. To include open commitments in
 the job-to-date cost amount on Job Cost reports and inquiries,
 select this checkbox. Otherwise, leave this checkbox clear.
This default may be overridden when entering
 selections for the report or inquiry.

Show projected cost?
If this checkbox is selected, the system by
 default will print reports in the format that compares actual cost
 to projected cost figures. If this checkbox is left clear, the
 default will print reports in the format that includes only actual
 costs and estimates.
Projected cost is determined through Projected
 Cost Entry, a screen that combines periodic feedback from the field
 regarding completion progress with actual cost-to-date information
 stored in the computer. In contrast, estimates represent planned
 costs recorded phase-by-phase, and might be less frequently adjusted
 during the progress of the job, except through change orders.
This entry serves as a default only when printing
 reports, and can be overwritten to print the other format as
 needed.

Include A/P invoices in progress?
Select this checkbox if you normally include
 unapproved A/P invoices. Otherwise, leave the checkbox clear if you
 want to include only approved invoices. This is the default used in
 inquiry and reports screens, but you may override this setting when
 entering selections for the report or inquiry.
When unapproved invoices are included in Job Cost
 inquiries or in Build Projected Costs, the unapproved amount will
 include any associated sales or use tax amounts.

Include Payroll in progress?
If this checkbox is selected, the system default
 will print reports with Pre-Time Card Entry information. If this
 checkbox is left clear, reports will print without Pre-Time Card
 Entry information.
This selection will serve as a default only when
 printing reports and can be overwritten.

Include Purchase Order quantities received?
If this checkbox is selected, the system will
 print reports including P/O Packing List Quantity Entry receivings
 by default. If this checkbox is left clear, reports will print
 without receivings.
This entry will serve as a default only when
 printing reports, and can be overwritten to print a different format
 as needed.

Show group subtotals?
If this checkbox is selected, the system by
 default will print reports in the group subtotal format. If this
 checkbox is left clear, the default will not print group subtotals
 on the reports. Group subtotals are determined phase by phase, based
 on the major and minor group definitions established in the Job Cost
 Installationscreen.
This entry will serve as a default only when
 printing reports, and can be overwritten to print the other format
 as needed.

Group summary
Select one of the following options (Major group
 only,Through minor group, or Phase summary) as
 the default group summary option to be used in the inquiry screens
 and reports. This default may be overridden when entering selections
 for the report or inquiry.

Period type
This selection will determine the default on Job
 Cost reports and inquiries (where this selection is available).
 Select to show
 Week-to-date amounts; Month-to-date
 amounts; or Year-to-date
 amounts. This default may be overridden when entering selections for
 the report or inquiry.

Productivity report ratio
Select either Hours per
 quantity or Quantity per
 hour. In the Job Phases screen, there are both
 Quantity and Hoursfields. The
 Quantityfield is related to the unit of measure
 fields. Hours and quantities entered here are then used in the Job
 Productivity Report and the Job Productivity Inquiry screens. You
 have the choice of which ratio to print on the report and inquiry
 screens. The selection specified here will default onto the starting
 screen for the Job Productivity Report, but may be overwritten when
 the report is run.
