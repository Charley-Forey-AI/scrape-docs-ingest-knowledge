# Reclassify Job Cost History: Transaction Type PR - Employee - Cost Center Information

If the cost center feature is enabled in the Enterprise Installation screen, for PR transaction type entries Spectrum allows you to edit a transaction for an employee only if you have permission to access that employee's information. Spectrum compares the employee's list of shared cost centers with cost centers in your assigned scheme; if there are no common cost centers, then transaction entry for that employee is not allowed. Validation also occurs if you attempt to assign an invalid employee to a transaction line.
When your cost centers scheme includes override settings for employee entries in Cost Center Scheme Maintenance, this screen will validate the cost center assigned to transaction detail lines based on these overrides. The override cost center(s) supersede the cost center list defined for the scheme in general. Spectrum compares the cost center assigned in the entry screen detail with employee override cost centers in your assigned scheme; if the cost center is not included, then the transaction entry line will not be allowed.
