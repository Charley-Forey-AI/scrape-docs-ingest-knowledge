# Contract Renewal Management - Field Descriptions

Use the table below for reference when completing the fields on this screen.
Field
Description

Expire date
The end date of the old contract will
 display in this field. no entry is allowed.
Site
The site code and site name displays in
 this field. No entry is allowed.
Old contract #
The old contract # and description
 displays in this field. No entry is allowed.
New contract #
The new contract # will default if
 determined during the build. Enter a new contract #, or leave this field blank
 and the contract will not be updated.Note: If
 you do not wish to renew a contract, use the Remove button to remove the
 contract record from the grid.

Old price
The total contract amount of the
 expiring contract will display in this field. No entry is allowed.
New price
The new contract amount will default if
 determined during the build, but can be changed.
Proposed?
Select this checkbox if the contract
 status is to be 'Proposed'. This checkbox will be selected by default if
 specified during the build, but can be de-selected.
Proposal date
If the contract status is 'Proposed'
 specify a proposal date for the new contract. This date will default if
 specified during the build, but can be changed.
'Honor for' days
If the contract status is 'Proposed'
 specify the number of days the proposal will be honored. This number will
 default if specified during the build, but can be changed.
New start date
The new contract start date will
 default by adding one day to the end date of the expiring contract. No entry is
 allowed.
New end date
The new end date will default by adding
 the same number of days in the expiring contract to the new start date. No
 entry is allowed.
Customer
The customer code and customer name
 will appear in this field. No entry is allowed.
Contract type
The contract type and description will
 display in this field. No entry is allowed.
Work order type
The work order type and description
 displays in this field. No entry is allowed.
