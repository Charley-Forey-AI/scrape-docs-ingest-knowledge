#  Employee FMLA Report - Field Descriptions

Use the table below for reference when completing the fields on this screen.
Field/Button
Description

Entity
If cost centers and entity tracking is enabled in Enterprise Installation, you can enter a cost center entity here in order to display entity-specific FMLA details.

Employee code
Enter the employee code to print, or press Enter to include ALL employee codes on the report.
Note: An employee must have a hire date in the Payroll > Employee Maintenance > Properties window in order for the employee's FMLA information to
 display on the Employee FMLA Report.

Social security number
Enter the social security number to include on the report, or press Enter to accept the software default of ALL.

Department
Enter the department code to include on the report, or press Enter to accept the software default of ALL.

Supervisor
Enter the supervisor code to include on the report, or press Enter to accept the software default of ALL.

Location
Enter the location code to include on the report, or press Enter to accept the software default of ALL.

Status
Enter the employee status code to include on the report, or press Enter to accept the software default of ALL.

Hire date
Enter first and last hire dates to include on the report, or press Enter to accept the default of the earliest hire date and end with the current Payroll processing date .

Include only employees currently on FMLA leave?
Select this checkbox if you want only employees who are currently on FMLA leave to display on the report.

Include history detail?
Select this checkbox to include employee's complete FMLA leave history. If you do not select this checkbox, the Employee FMLA Report will only consist of the employee's leave totals (weeks remaining, weeks used, and so forth) without providing detailed information about the employee's previous FMLA leave.

FMLA start date
If you selected the Include history detail checkbox, enter the earliest and last FMLA leave dates to include on the report, or press Enter to accept the current Payroll processing date. Based on the date you enter, select history records for the date range will appear on the report for all employees (even those employees who haven't taken any leave).
