# Recurring Benefit Code Maintenance - Field Descriptions

Use the table below for reference when completing the fields on this screen.
Field/Button
Description

Benefit code
Enter the benefit code you want to add. Depending on your level of security
 within the software, you may be able to access the Payroll
 Deduction/ Add-on Code Maintenance screen by clicking the
 Maintenance button in the Search
 Benefits window. The Deduction/Add-on Code
 Maintenance screen is used to maintain Payroll
 deduction/add-on codes.

Code type
The benefit code type, if a description exists, will display.

Benefit start date
Enter the benefit start date, or press Enter to accept the software default. Options include:

- Upon eligibility

- First of the next month

- First of the next quarter

- First of the next calendar year

- First of the next fiscal year

The benefit start date filter assigned to the benefit code currently being
 defined determines when employees will become eligible for benefits in the
 Authorize Recurring Benefits screen.

Matching code
Enter a matching payroll deduction code that will be used to record a second add-on or deduction code that may be assigned in conjunction with the benefit code being defined. For example, when a 401(k) benefit is set up, the corresponding employer matching code can be specified here if the codes being specified share the same eligibility requirements.
Note: The matching code selected in this field cannot be a benefit
 code already set up in this screen. When an employee is eligible for two
 benefit codes, for example, a 401(k) deduction and an employer-matched
 contribution, this field is used to simplify the process by setting these
 two codes up simultaneously.

Home department
Enter the benefit code's home department code, or press Enter to accept the software default.

Employee status
Enter the employee status code associated with this benefit code, or press Enter to accept the software default.

Pay type
Use the drop-down menu to select an employee pay type to determine
 eligibility, or press Enter to accept the software
 default. Drop-down menu options include:

-  All

-  Commission

-  Hourly

-  Overtime Salary

-  Salary

Part-time
Use the drop-down menu to specify whether or not employee's with casual status (part-time) are eligible for the selected benefit.

Minimum standard hours
Enter the minimum number of hours needed for this benefit code to take effect.

Minimum days since hire date
Enter the minimum number of days since hire for the specified benefit code.

Minimum hours worked since hire date
Enter the minimum number of hours worked since hire date for the specified
 benefit code. This field is used to calculate the selected benefit using
 hours from Payroll time card history with work dates that fall between the
 hire date and the Calculate eligibility as of field in the header section of
 the Authorize Benefit Management and
 Benefit Management screens (or the Payroll
 processing date for calculations done in the
 Employees screen).
 If this field is left blank (indicating that there is no eligibility requirement), then dates in this field are not relevant.
Note: In order to calculate this field, a hire date must be
 entered in Employees > Main Properties.

User-defined eligibility
Use the user-defined eligibility fields to specify benefit requirements
 according to your company's needs. Select an eligibility limitation to be
 applied to the selected benefit code. If a date or numeric user-defined code
 is selected, then the Numeric User-Defined Fields
 window displays, prompting you to set calculation limitations and values for
 determining the user-defined eligibility requirement.
If you do not select information in this field, then it will not be used to determine eligibility.
