# FMLA Weeks Used Update - Field Descriptions

Use the table below for reference when completing the fields on this screen.
Field/Button
Description

Entity
If cost centers and entity tracking is enabled in Enterprise Installation, you can enter a cost center entity here in order to calculate entity-specific FMLA usage.

Employee code Social security number Department Supervisor Location Status
Enter the employee code, social security number, department code, supervisor code, location code, and employee status code to include on the report, or press Enter to include ALL codes on the report.

Hire date
If you want to perform the FMLA update based on employment dates, enter the earliest and last hire dates to update, or press Enter to accept the current Payroll processing date.

Include employees who are not currently on FMLA leave?
Select this checkbox to include all employees in the update, regardless of whether or not they are on FMLA leave. Otherwise, leave this checkbox clear to update only employees currently on FMLA leave. All employees who are not eligible for FMLA are always excluded from this update.

12 month period is based on
This field reflects the selection made in 12 month period is based on field in
 the Human Resources Installation > FMLA tab. No entry is allowed.
This field is unavailable if the A rolling 12-month period measured backward from FMLA leave
 option is selected on the Human Resources Installation > FMLA tab.

Calculate as of date
Enter the date to calculate the FMLA leave through, press F4 to select a date from the Date Change window, or press Enter to accept the current processing date.
Note: The system will round up the amount of leave taken. For
 example, if an employee takes two weeks and one day worth of leave, the
 software will display three weeks on the report.

Start date
Enter the start date of the current 12-month period, or press Enter to accept the current processing date.
This field displays only if one of the following options is selected in the
 12 month period is based
 on field on the Human Resources Installation > FMLA tab:

- Calendar year

- A fixed 12-month period such as fiscal year

If the Calendar year
 or the A fixed 12-month period
 such as fiscal year option is selected on the Human Resources Installation > FMLA tab, the date in the Start date defaults to the
 date that is one year (minus one day) earlier than the date in the
 Calculate as of
 date field. For example, if you enter 09/24/02 in the
 Calculate as of date field, then 09/25/01 displays
 in the Start date field.
